## Goal for this iteration

M11.1b: relax `validateProjectFileName` to permit `/`-separated
multi-segment paths (each segment satisfies the existing
single-segment rule; no empty/`.`/`..` segments, no leading or
trailing slash). The blob store already handles slashed keys
(`LocalFsBlobStore.put` does `mkdir(dirname, recursive)`), and
`fileTree.buildFileTree` already groups multi-segment paths into
a forest. Once the validator stops rejecting them, slashed names
flow end-to-end. Also: have `LocalFsBlobStore.delete` reap empty
parent directories (best-effort) so disk state stays tidy after
the last file in a virtual folder is removed.

M11.1b is the prerequisite for M11.3 (virtual-folder create). It
does not enable any UI yet — the rename/create affordances still
trim and submit a single string; entering `chapters/intro.tex`
just stops being rejected. Tree rendering follows automatically.

Live blocked: M15 still waits on
`vendor/supertex/discussion/764_answer.md`. M13.2(b).5 R1
blocked on shared blob store binding. Both checked at the start
of the iteration; no movement.

## Approach

1. Rewrite `validateProjectFileName` in `packages/protocol/src/index.ts`
   to validate per-segment.
2. Update protocol codec test to cover new accept/reject shapes.
3. Add `LocalFsBlobStore.delete` parent-reap (rmdir loop, stop at
   root, swallow ENOTEMPTY/ENOENT).
4. Add a sidecar persistence smoke that round-trips a
   `chapters/intro.tex` add → list → rename → delete through a
   real `LocalFsBlobStore`.
5. Update `fileTree.ts` head-comment (no longer "server rejects
   slashes").

## Actions

- `packages/protocol/src/index.ts` — rewrote `validateProjectFileName`
  per-segment. Renamed the internal `FILE_NAME_RE` to
  `FILE_NAME_SEGMENT_RE`. New reject categories:
  `name must not start or end with '/'`, `empty segment`,
  `reserved segment` (replaces `reserved name`).
- `packages/protocol/test/codec.test.mjs` — added three accept
  cases (`chapters/intro.tex`, `chapters/sub/deep.tex`,
  `a/b/c/d/e.tex`) and five new reject shapes covering each
  branch.
- `packages/blobs/src/localFs.ts` — `delete()` now walks up
  parent directories calling `rmdir` and stops at the first
  `ENOTEMPTY`/`EEXIST`/`ENOENT` or the configured root.
- `packages/blobs/test/localFs.test.mjs` — added a parent-reap
  case: `a/b/c/leaf.tex` deletion reaps `b` + `c` but leaves `a`
  (sibling file pinning it); subsequent sibling delete reaps `a`
  but never the store root.
- `apps/sidecar/test/slashedFileNames.test.mjs` (new) — full
  round-trip through `createProjectPersistence` with a real
  `LocalFsBlobStore`: addFile of slashed paths PUTs through
  `mkdir -p`; `listProjectFiles` surfaces them; fresh hydration
  loads each into its own `Y.Text`; rename across folder
  boundaries works; deleting the deepest leaf reaps the virtual
  folder; bad shapes (`../evil`, `a/`, `a//b`) are rejected at
  the `addFile` boundary. Wired into `test_node_suites.py`.
- `apps/sidecar/test/serverRenameFile.test.mjs` — the rejection
  case using `bad/name` is now valid; swapped for `bad name`
  (space) to keep the 4-rejection invariant.
- `apps/web/src/lib/fileTree.ts` — head-comment updated; the
  forest already materialises folder nodes from slashed input.
- `.autodev/PLAN.md` — M11.1b moved into the landed list with a
  pointer to the three locks; M11.3 dependency line updated.

  UI surfaces a way to type one yet. `FileTree.svelte`'s new-file
