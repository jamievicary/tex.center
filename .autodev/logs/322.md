## Goal for this iteration

`N=322` ordinary iteration (no `%10` obligation). The 321 plan-review
log named M20.2 as the next ordinary work, with the sub-split:

- (a) wire the binding + interface end-to-end with no rehydration
- (b) wire the per-settle persist write
- (c) wire cold-boot rehydration + flip the SSR-seed-gate condition

This iteration takes the (a) scaffolding step as **dark code** (per
the multi-iteration-milestone rule): give the web tier the same
`BlobStore` capability the sidecar has, exposed via a thin
`webBlobStoreFromEnv()` helper, with a `coldSourceFor(blobStore,
projectId)` primitive that reads the persisted `main.tex` blob.
**Not wired into the runtime path yet** — the M15 `seedDocFor` chain
(db `seed_doc` → Machine env) remains the only seed source, so
production behaviour is unchanged. The next iteration takes
M20.2(c): compose `coldSourceFor` into `seedDocFor` as the
preferred lookup (blob beats db beats hello-world). Persistence side
(b) is already implemented today by `persistence.maybePersist()` on
every `runCompile` and by `persistAllCheckpoints()` on idle-stop —
what's missing for live cold-boot rehydration to work *across*
Machine lifetimes is genuinely shared backing storage (Tigris/S3),
which is its own scaffolding step yet to come.

## Approach

- Add `@tex-center/blobs` workspace dep to `apps/web/package.json`.
- New file `apps/web/src/lib/server/blobStore.ts` exporting
  `webBlobStoreFromEnv()` (a thin wrapper around the
  `defaultBlobStoreFromEnv()` already in sidecar persistence — same
  env-var protocol so a single deploy config bakes both sides) and
  `coldSourceFor(blobStore, projectId)` that returns the persisted
  `main.tex` source (or `null`).
- The sidecar's `defaultBlobStoreFromEnv()` + `mainTexKey` /
  `projectFilesDir` / `projectFileKey` / `projectCheckpointKey`
  key builders today live in `apps/sidecar/src/persistence.ts`.
  Lift the three pieces the web tier needs — the env selector,
  `mainTexKey`, and the shared `MAIN_DOC_NAME`-backed file-key
  helpers `projectFilesDir`/`projectFileKey` — into
  `@tex-center/blobs` itself (new files `envSelect.ts` and
  `keys.ts`). This is the natural home (the blobs package already
  owns `validateKey` and `LocalFsBlobStore`) and removes the
  alternative of either importing across `apps/*` or maintaining
  two parallel copies of identical 15-line env-selector logic.
  `projectCheckpointKey` stays in sidecar persistence — the web
  tier has no reason to know the checkpoint key shape.
  Sidecar persistence re-imports from `@tex-center/blobs` and
  the type signatures it exposes externally are unchanged
  (`createProjectPersistence`, etc.).
- Test file `apps/web/test/blobStore.test.mjs` exercising both
  helpers against a `LocalFsBlobStore` in `mkdtemp`.
- Register the new test in `tests_normal/cases/test_node_suites.py`.

## Notes for future iterations

- **M20.2(c) cutover** is now a one-line wiring change at
  `apps/web/src/server.ts:86-89` (the `seedDocFor` lambda). The
  natural shape: try `coldSourceFor(webBlobStoreFromEnv(), id)`
  first; on null fall through to `getProjectSeedDoc(db, id)`. The
  iteration that lands this should also add an integration test
  pointing apps/web and a synthetic sidecar at a shared
  `BLOB_STORE_LOCAL_DIR` so the cold-storage path is exercised
  end-to-end despite production still being per-Machine-local.
  The SSR seed-gate condition flip ("no machine assignment" → "no
  persisted blob") rides on the same iteration: once the seed
  source can ride on a blob (independent of any Machine), the
  gate's invariant must be "do we have a seed to render?" rather
  than "is there a Machine assigned?". The iteration after that
  is the S3/Tigris adapter — a substantial slice on its own; bake
  a request-recording HTTP stub so adapter tests are deterministic.
- **Two parallel env selectors avoided.** The `defaultBlobStoreFromEnv`
  lift collapses what would otherwise be a 15-line copy/paste pair
  between sidecar persistence and the new web blob-store module.
  Any future env-protocol change (new `BLOB_STORE` values,
  alternative credential plumbing) now happens in one place.
- The `webBlobStoreFromEnv` indirection over
  `defaultBlobStoreFromEnv` is intentional: same call shape today,
  but it's the seam for any web-only switch (e.g. read-only mode,
  prefix-scoping) that the sidecar shouldn't honour. If after a
  few iterations no such switch is needed, fold it away.
