## Goal for this iteration

Take the "wire sidecar deploy into CI" smaller-alternative
listed in iter 123's notes and in PLAN's Current focus. With
iter 123 flipping `SIDECAR_COMPILER=supertex-daemon` to the
production default in the sidecar Dockerfile, the live cutover
still requires a manual `flyctl deploy` against
`tex-center-sidecar`. A dedicated CI workflow makes future
sidecar changes — including this daemon flip — reach prod on
push to `main` the same way `tex-center` already does, and
removes "remember to redeploy the sidecar" as an ongoing
manual step. M7.4.2 (upstream serialise/restore wire) remains
the nominal next focus but lives in the supertex repo, out of
scope here.

## Approach

- New `.github/workflows/deploy-sidecar.yml`:
  - `on.push.branches=[main]` with `paths:` filter scoped to
    `apps/sidecar/**`, the `packages/*` workspace deps the
    sidecar pulls in (`blobs`, `db`, `protocol`),
    `vendor/supertex` (submodule pointer), `vendor/engine/**`
    (vendored ELF + provenance), the lockfile/workspace, and
    the workflow file itself. Routine PLAN/log churn doesn't
    trigger a deploy.
  - `workflow_dispatch:` so the daemon flip can be cut over
    manually without a code touch.
  - `concurrency.group=fly-deploy-sidecar` (distinct from the
    control-plane `fly-deploy` group so the two apps can roll
    out in parallel).
  - `actions/checkout@v5` with `submodules: recursive` because
    `apps/sidecar/Dockerfile` COPYs `vendor/supertex/` into
    the builder stage.
  - Canonical deploy command per PLAN (both `-a` and
    `--config`): `flyctl deploy --remote-only --no-public-ips
    -a tex-center-sidecar --config apps/sidecar/fly.toml .`.
  - Reuses the existing `FLY_API_TOKEN` repo Actions secret.
- New `tests_normal/cases/test_deploy_sidecar_workflow.py`
  mirroring `test_deploy_workflow.py`: parses the YAML and
  pins triggers (push-to-main + `paths` must include the
  workflow file itself + sidecar tree + engine tree;
  `workflow_dispatch` present), checkout-with-submodules,
  setup-flyctl, the canonical deploy flags (`--remote-only`,
  `--no-public-ips`, `-a tex-center-sidecar`,
  `--config apps/sidecar/fly.toml`), `FLY_API_TOKEN` wiring,
  and that the referenced `apps/sidecar/fly.toml` exists.
- PLAN Current focus rewritten to reflect that the cutover
  is now CI-driven, with the `gh workflow run` escape hatch.
