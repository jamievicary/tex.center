## Goal for this iteration

Land **M12 — draggable panel dividers** for the editor route, the
next sequenced milestone in `PLAN.md` after M13.2(b).3's spec
landed last iteration (and is expected RED until the seed-widening
follow-up). M12 is a self-contained UX feature; doing it now keeps
the plan moving while the M13.2(b) widening dependency (shared R2
binding) waits.

## Approach

Three columns, two dividers. Tree and preview hold absolute px in
CSS custom properties; the editor pane takes the remaining `1fr`.
Pointer-capture drag updates the variables live; on `pointerup`
the widths are persisted to `localStorage` keyed by
`editor-widths:${projectId}`. Min widths defended on every drag so
the editor pane can never collapse below `MIN_EDITOR_PX`. On
window resize, widths are reclamped (no proportional rescale —
deliberate choice; user re-adjusts if needed).

Default state (no localStorage entry): tree=220px, preview≈half of
the remaining viewport (computed from the live shell width on
mount). Default keeps the pre-M12 visual ratio close enough.

CSS grid uses 5 columns: `var(--col-tree) 4px 1fr 4px
minmax(0, var(--col-preview))`. The named grid areas
`tree dtree editor dpreview preview` keep the existing test
selectors (`.tree`, `.editor`, `.preview`) untouched. Dividers
carry `data-divider="tree"` / `data-divider="preview"` for test
selection.

Deferred from M12 (single-iter scope guard): "file picker
collapsible to zero with a re-open chevron" — adds modal logic
on top of the basic drag mechanic. The min-width clamp (150px on
tree) keeps the current iteration's behaviour safe and the
collapse-to-zero affordance is a future-ideas-worthy follow-up.
