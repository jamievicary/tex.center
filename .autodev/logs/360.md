N=360, refactor iteration (`N % 10 == 0`). Surveyed `src/`,
`tests_normal/`, `tests_gold/` for the single highest-leverage
improvement. Top candidate: the two GT-6-live-editable gold specs
(`verifyLiveGt6LiveEditableState.spec.ts` and
`verifyLiveGt6LiveEditableStateStopped.spec.ts`) are >290 lines
each, differ only in which Fly API verb drives the per-project

polled state string, and three timing/label constants. `diff`
between them is almost entirely substitutions of those tokens —
the WS counter wiring, cold-start hand-off, dashboard-reclick
sequence, three-phase diagnostic stdout line, and the

`clickToWsOpenMs` / `clickToFirstFrameMs` / `wsPostClick=opens:N/
closes:M` diagnostic to both). That's the textbook signal for
"these want to be one helper" — and the cost of leaving it is
real: any further iteration on either spec doubles the
edit/review/regression-risk burden.

Coherent slice: extract a shared helper
`runColdFromInactiveLiveEditableTest(options, ctx)` under
`tests_gold/playwright/fixtures/coldFromInactiveLiveEditableTest.ts`;
both spec files become thin wrappers that supply the variant
params (label, flyAction, flyState, settleTimeoutMs,
projectNamePrefix) and own their own `testInfo.setTimeout` budget
+ skip predicates. No assertion changes, no diagnostic format
changes — Playwright sees the same test IDs and emits the same
stdout shape.

## Why a single helper, not a factory of `test()` calls

The natural alternative is `defineColdFromInactiveSpec(...)` that
internally calls `test.describe(...)` / `test()`. Rejected
because:

- Each variant has its own per-test budget (40 s vs 60 s) and a
  bespoke top-of-file narrative comment block tying it to the
  M13.2(b).3 vs (b).4 PLAN entry. Keeping those at the spec level
  preserves human-readable file-level intent.
- Skipping behaviour and `testInfo.setTimeout` are spec-local
  concerns; folding them into the factory would obscure that
  M13.2(b).4 is the looser-budgeted variant.
- Playwright tooling (filter-by-file, `--update-snapshots`, log
  output) is friendlier with per-variant top-level spec files.

The helper extracts only the body that is genuinely identical.
