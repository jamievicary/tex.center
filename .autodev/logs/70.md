## Notes for future iterations

- The shape `{ verb: true } | { verb: false; reason: string }`
  used by `addFile` / `renameFile` / `deleteFile` is awkward —
  the helper has to cast through `Record<string, unknown>`
  because each verb's success branch literally has no shared
  properties. A unification to `{ ok: true } | { ok: false;
  reason: string }` would let the helper be type-clean and ditch
  the cast; one signature change in `persistence.ts` plus a
  handful of test assertions. Not done this iteration to keep the
  refactor scope-tight, but worth doing next time someone touches
  the file-op surface.
