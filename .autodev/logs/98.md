## Goal for this iteration

M8.pw.2 — encode the manual `deploy/VERIFY.md` probes as a
Playwright spec so deploy-touching iterations get a single
command to verify a fresh control-plane deploy end-to-end.

## Approach

- Add `tests_gold/playwright/verifyLive.spec.ts` containing the
  five probes (healthz body, `/`, OAuth start, two WS upgrade
  paths).
- Self-skip when `testInfo.project.name !== "live"` so the
  default local gold run pays no cost.
- Use `playwright`'s `request` API for the three HTTP probes
  (`maxRedirects: 0` for the OAuth start so the 302 is
  preserved). Use Node's `https.request` directly for the two
  WS upgrade probes so the upgrade response status code is what
  gets asserted (Playwright's request fixture isn't a WS
  client).
- Update `deploy/VERIFY.md` with the canonical one-liner and a
  pointer to the `TEXCENTER_LIVE_TESTS=1` gold wrapper.
