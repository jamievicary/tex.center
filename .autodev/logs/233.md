## Approach

Followed the design in `231_answer.md` verbatim:

- Switch the fixture import from `sharedLiveProject` to
  `authedPage` (so the spec gets the `db` worker fixture and
  no longer reuses the warm `liveProject`).
- Mint a fresh `projects` row via `createProject(db.db.db, …)`
  exactly like GT-8. Seed template materialises with the
  `documentclass` sentinel; no Machine has been assigned, so
  this is a cold-sidecar path on every invocation.
- `goto("/projects")`, `locator('a[href="/editor/<id>"]')`,
  `click()`, then `waitForURL("**/editor/<id>")` +
  `waitForLoadState("domcontentloaded")` to mark "editor
  route interactive" as the bound's start point.
- 500 ms `expect.poll` for `.cm-content` to contain
  `documentclass`. On timeout, continue polling (without the
  bound) up to a 60 s diagnostic deadline so the failure
  message reports the actual appearance time — invaluable
  for triage.
- `finally` block: `cleanupProjectMachine` against the freshly-
  assigned Machine (if any), then `delete projects where id`.
  Mirrors GT-8's teardown.
