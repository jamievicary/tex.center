## Goal for this iteration

M8.pw.0 — Playwright skeleton. Land the infrastructure to author
and run browser tests so the next control-plane redeploy
(M7.0.3) has a programmatic UI-level acceptance signal (iter-73
HTTP probes missed the iter-76 OAuth bug — see discussion 77).
This iteration lands the workspace devDep, idempotent
DrvFs-aware Chromium provisioner, two-project Playwright config,
one trivial DOM-level GOAL-criterion-#1 test, and the Python
gold-runner integration. The `live` target is wired but gated
on `TEXCENTER_LIVE_TESTS=1` (skipped by default so production
isn't beaten on every iteration).

## Approach

Mirror the existing `setup_node.sh` pattern for the browser
provisioner. Workspace devDep at the root so all packages share
the same `@playwright/test` version (matches the
typescript devDep placement). Use Playwright's built-in
`webServer:` block for the `local` target (Playwright manages
dev-server lifecycle), and disable it for `live` via a runner-set
`PLAYWRIGHT_SKIP_WEBSERVER=1` env so the config stays a single
file.
