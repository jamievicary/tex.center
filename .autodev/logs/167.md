## Implementation summary

Sidecar-side seeding (rather than control-plane), because the
web app has no blob-store wiring today *and* `BLOB_STORE` isn't
configured on the live sidecar either — only the sidecar's
`createProjectPersistence` is a single chokepoint covering both
the in-memory and blob-backed paths.

- `packages/protocol/src/index.ts`: new `MAIN_DOC_HELLO_WORLD`
  constant (the 4-line template, exact bytes).
- `apps/sidecar/src/persistence.ts`: in-memory branch seeds
  `Y.Text(main.tex)` on construction iff empty; blob-backed
  branch seeds + persists iff no `main.tex` blob exists.
  Existing-but-empty blobs are NOT clobbered.
- `apps/sidecar/test/persistenceSeed.test.mjs`: five-case
  drift-lock test (bytes equal canonical template, four
  behavioural cases).
- `tests_normal/cases/test_node_suites.py`: wires the new test.
- `apps/sidecar/test/server.test.mjs`: updated for new contract
  — client now syncs the seeded state via incoming doc-update
  frames before appending a user edit (avoids two concurrent
  inserts-at-0 with clientID-dependent merge order).
