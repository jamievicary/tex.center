## Goal for this iteration

**Discussion mode.** `.autodev/discussion/129_question.md` reports a
production-down: `https://tex.center/auth/google/callback` returns 500
after Google consent. The question (correctly) demands treating this as
a P0 over PLAN's M7.4.2 work, asks to diagnose before patching, and
explicitly forbids "make the symptom go away" workarounds.

Answer file: `.autodev/discussion/129_answer.md`. Diagnosis pivot from
the question's leading hypotheses: not a DB issue
(`/readyz` reports `db: up`), not the iter-95 entry wrapper (SvelteKit's
default 500 page renders, so errors *are* propagating). The actual
root cause, surfaced by `flyctl logs --no-tail -a tex-center`, is
`ERR_MODULE_NOT_FOUND: Cannot find package 'jose' imported from
/app/build/server/chunks/_server.ts-*.js` — adapter-node leaves
`jose` (used by `googleTokens.ts` for Google JWKS verify) as an
external bare specifier, but `apps/web/Dockerfile`'s runtime stage
ships no `node_modules`. Latent since the first auth-bearing deploy:
all other live probes avoid `jose`'s import graph; workspace deps
get inlined because they're symlinks.

Discussion mode allows fix code in the same iteration when it
directly addresses the question — and the question explicitly asks
for a real root-cause fix here, not a follow-up commitment. Landing
the fix + regression test together this iteration.
