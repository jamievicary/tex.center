## Goal for this iteration

Plan's "next ordinary iteration" (M6.3.1) is out-of-tree. File-tree
CRUD + text upload is now complete. The next in-tree groundwork
slice that doesn't need docker (S3) or a streaming compiler
(awaitPdfStable wiring) is the `projects` query layer. The
`projects` table already exists in DDL + Drizzle (iter 16‚Äì17),
but no helper functions exist for inserting / fetching / listing
project rows ‚Äî every codepath today uses the hardcoded literal
`"default"` for the project id. Add `packages/db/src/projects.ts`
with `createProject`, `getProjectById`, `listProjectsByOwnerId`
mirroring the shape of `users.ts` / `sessions.ts`, and a paired
PGlite gold test. This is the storage primitive a future
"projects dashboard" slice will sit on top of (GOAL.md item 5,
acceptance #6).

## Approach

- New file `packages/db/src/projects.ts` exporting:
  - `createProject(db, { ownerId, name }): Promise<ProjectRow>` ‚Äî
    insert with a fresh uuid; rely on DB defaults for timestamps.
  - `getProjectById(db, id): Promise<ProjectRow | null>` ‚Äî null
    on miss, mirroring `getSessionWithUser`'s shape.
  - `listProjectsByOwnerId(db, ownerId): Promise<ProjectRow[]>` ‚Äî
    sorted by `created_at` then `id` (deterministic for tests).
- Re-export from `packages/db/src/index.ts`.
- New `packages/db/test/projects-pglite.test.mjs` mirroring
  `users-sessions-pglite.test.mjs`: insert + fetch round-trip,
  list-by-owner across two owners, FK-enforcement when ownerId
  doesn't exist, miss-by-id returns null.
- New gold runner `tests_gold/cases/test_pglite_projects.py`.
- No web/sidecar wiring this iteration ‚Äî future slices route
  `/editor/<projectId>` and the dashboard listing.
