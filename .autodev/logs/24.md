## Approach

1. New `packages/blobs` (workspace package, mirrors the layout
   of `packages/protocol`): `package.json`, `tsconfig.json`,
   `src/index.ts`, `src/key.ts`, `src/localFs.ts`.
2. `BlobStore` interface: `get`/`put`/`list`/`delete`,
   `Uint8Array` payloads, forward-slash key strings.
3. `LocalFsBlobStore` maps key segments onto a directory tree
   under a caller-supplied root. Writes go through `<f>.tmp` +
   rename for atomicity; `list(prefix)` walks the tree in lex
   order; `.tmp` survivors from a crashed put are skipped.
4. Shared `validateKey` rejects empty / absolute / trailing-
   slash / `.` / `..` / NUL / backslash and any segment outside
   `[A-Za-z0-9._-]` â€” same guarantees the future `S3BlobStore`
   will inherit.

   overwrite atomicity check, directory-prefix list, mid-segment
   prefix list, empty-prefix, idempotent delete, traversal
   rejection). Registered in
   `tests_normal/cases/test_node_suites.py`.

   blocked on MinIO), M4.3.2 (sidecar wiring); update Current

## Notes for future iterations

- `LocalFsBlobStore.list("")` walks the entire root; fine for
  tests but worth replacing or adding a depth bound if a real
  workload ever uses it.
- `S3BlobStore` should reuse `validateKey` so prod and dev
  reject the same set of bad keys; once both adapters exist,
  consider lifting validation into a shared base class.
- The atomic-write guarantee assumes `rename(2)` is atomic on
  the underlying filesystem. WSL2's drvfs (`/mnt/c`) is fine for
  tests because temps live under `os.tmpdir()` (ext4); a
  production path that ever points the local adapter at drvfs
  would need re-checking.
