## Goal for this iteration

M17.preview-render — eliminate the PDF preview flash on every
recompile, replace with a per-page cross-fade. Priority-queue
next-up after the M15 sidecar fix shipped as v39 (operator-cadence
deploy completed during iter 270; verified via `flyctl releases`
this iter).

## Approach

- New module `apps/web/src/lib/pdfFadeController.ts` owns the
  per-page state machine. DOM-free: speaks to a `FadeAdapter`
  interface so the state-machine half is unit-testable under tsx
  without jsdom.
- `PdfViewer.svelte` now (1) renders every PDF.js page off-DOM
  (await `page.render` into a detached canvas) (2) then hands the
  controller a `CanvasDescriptor[]` for a single synchronous
  commit. Per-page wrapper `<div class="pdf-page">` carries the
  intrinsic page geometry via `width` + `aspect-ratio` and stacks
  the two canvases absolutely during the cross-fade.
- `data-page` moved to the wrapper, not the canvas — IO targets a
  stable DOM node across renders, so the old `tracker.reset()`
  on every commit is gone.
- Mid-fade interrupt: a fresh commit calls
  `commitFadeImmediately` per in-flight page first — drops the
  leaving canvas, snaps the entering canvas to opacity 1 — then
  starts the new cross-fade with that settled canvas as the new
  "leaving" element.
- Wrappers added (page count grows) fade in their opacity from 0
  → 1; wrappers removed (page count shrinks) fade out 1 → 0 then
  detach on `transitionend`. Geometry change between commits
  transitions `width` + `aspect-ratio` over 180 ms.

## Problems encountered

- `aspect-ratio` transitions are CSS-supported but visual
  smoothness depends on the browser honouring numeric transitions
  on the property. Chromium does; the cross-fade still works if
  it doesn't (the wrapper just snaps to the new aspect on commit
  rather than easing). Not load-bearing.
- The "canvas + canvas" stacking initially relied on DOM order
  (second canvas absolute, first flowed). Switched to "both
  canvases absolute, wrapper carries geometry" so the
  swap is order-independent — appending vs prepending the new
  canvas no longer matters.
