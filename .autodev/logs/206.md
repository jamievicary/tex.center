helper (`apps/web/src/lib/editorMarks.ts`) with named constants

(`editor:route-mounted`) in the editor page's `onMount`. This is
deliberately dark-code scaffolding (per engineer.md §3) for a

in progress at iter-206 start); GT-B/C/D/5 verification waits on
the next harness gold pass. This iteration picks an independent
post-MVP slice that does not touch the sidecar, so the in-flight
deploy is undisturbed.

## Actions

- New `apps/web/src/lib/editorMarks.ts`:
  - Name constants `EDITOR_ROUTE_MOUNTED`, `EDITOR_WS_OPEN`,
    `EDITOR_YJS_HYDRATED`, `EDITOR_FIRST_TEXT_PAINT`,
    `EDITOR_FIRST_PDF_SEGMENT`, and a typed `EditorMarkName` union.
  - `markOnce(name, perf?)` records via `performance.mark` iff
    `getEntriesByName(name).length === 0`. Default `perf` falls
    back through `typeof performance !== "undefined"` so SSR /
    test environments without the Performance API are a silent
    no-op (returns `false`). Caller signal: boolean = "actually
    recorded".
- `apps/web/test/editorMarks.test.mjs`: covers first-call records,
  second-call no-op, distinct-name independence, and the
  no-Performance branch (simulated via `delete globalThis.performance`).
- `tests_normal/cases/test_node_suites.py`: wired as
  `test_web_editor_marks`.
- `apps/web/src/routes/editor/[projectId]/+page.svelte`: imports
  `EDITOR_ROUTE_MOUNTED` + `markOnce`, calls it at the top of
  `onMount` (before `initDebugFlag` so the mark records the
  earliest possible "editor route alive" moment).
