## Goal for this iteration

M22.2 — GT-F local Playwright. M22.4b closed iter 317; PLAN
priority queue puts GT-F next. Lock the locally-testable surface
of the debug-toast contract: `?debug=1/0` URL flag → persisted
`localStorage["editor-settings"].debugMode`, the settings popover
checkbox is the same single source of truth, and Ctrl+Shift+D
toggles the same key.

Wire-driven assertions ("single keystroke → green Yjs-op toast",
"after compile a blue pdf-segment toast") cannot run locally: the
local stack has no sidecar, the editor's `+page.svelte` gates
CodeMirror mount on `snapshot.hydrated`, the WS never opens, and
`WsClient.onDocUpdate` only fires the `outgoing-doc-update` debug
event when `send()` succeeds (line 127 — gated on
`socket.readyState === OPEN`). The unit suites
`apps/web/test/wsClientDebugEvents.test.mjs` and
`apps/web/test/debugToastsToggle.test.mjs` already lock the wire
event → toast mapping behind `?debug=1`. The locally-untestable
part lives on live and is exercised end-to-end by the existing
GT-3/GT-4 typing specs (they don't currently *assert* toasts, but
the toast wiring is on the same code path that's already green
there).

## Approach

New spec `tests_gold/playwright/editorDebugToggle.spec.ts`,
local-only (skip on live — the live spec would race with the
sidecar's own debug events colliding into the toast store). Four
cases, each in a fresh `authedPage` context (localStorage is
empty at start):

1. `?debug=1` visit persists `debugMode: true` to localStorage
   (default is already true, but the flag forces the persist path
   through `initDebugMode`'s `shouldPersist: true` branch). Open
   popover, assert checkbox `checked`.
2. `?debug=0` visit persists `debugMode: false`. Open popover,
   assert checkbox not checked.
3. Click the popover checkbox after a `?debug=1` visit; assert
   localStorage flips and the checkbox state mirrors.
4. Ctrl+Shift+D keyboard shortcut toggles debugMode whether or
   not the popover is open.

All four wait on `getByTestId("settings-cog")` visible before
reading localStorage, which gates on the editor route's onMount
firing (and therefore the `initDebugMode` + `persistSettings`
calls completing). No reliance on WS, hydration, or editor mount.

Each case also delete-cascades its seeded project in `afterEach`
to match the surrounding spec convention.

PLAN update: M22.2 closes with the new spec as its lock; GT-F's
wire-driven assertions get demoted to a "live spec follow-up"
note rather than staying open under M22.2. The M9.editor-ux GT-F
entry rewords accordingly.
