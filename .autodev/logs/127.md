## Goal for this iteration

Pick up FUTURE_IDEAS bullet "`/readyz` endpoint with backing-service
status" for the control plane. `apps/web` currently only exposes
`/healthz` (liveness, intentionally decoupled from Postgres). Add a
sibling `/readyz` route that probes the DB via `SELECT 1` and reports
`{ ok, protocol, db: { state, error? } }` with HTTP 503 on probe
failure. Matches the sidecar `/healthz` shape and unblocks future
deploy-verification probes that want to gate on backing-service
readiness.

M7.4.2 (upstream supertex serialise/restore wire) remains the
nominal PLAN focus but lives in the supertex repo and is too big
for one local iteration. This is a clean local slice with clear
test coverage that retires a FUTURE_IDEAS bullet.

## Approach

- Pure helper `apps/web/src/lib/server/readyz.ts` exporting
  `probeReady({ probeDb })` where `probeDb` returns `null` (no
  handle configured) or a `Promise<void>` resolving on success /
  rejecting on failure. Result `{ ok, protocol, db }` with `ok`
  false when `db.state === "down"`.
- Route `apps/web/src/routes/readyz/+server.ts` wires the helper
  with `process.env.DATABASE_URL`-gated `getDb().client\`SELECT 1\``.
  Returns 503 when `!ok` so external probes can gate on it.
- Unit test `apps/web/test/readyz.test.mjs`: covers null (absent),
  resolving (up), rejecting Error (down), rejecting non-Error
  (string error stringified).
- Static parser test `tests_normal/cases/test_readyz_route.py`
  mirrors `test_healthz_route.py`: exports `GET`, prerender false,
  no-store, protocol marker, references the helper.
- Register the new mjs test in `tests_normal/cases/test_node_suites.py`.
- Remove the FUTURE_IDEAS bullet.
