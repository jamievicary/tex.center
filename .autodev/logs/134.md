## Goal for this iteration

M8.pw.3.1 ŌĆö land the test-only callback-finaliser route. With the
`finalizeGoogleSession` extraction in iter 133, this iteration adds:
the SvelteKit route `POST /auth/google/test-callback` (404 when

  `resolveTestCallback({ bodyText, bypassHeader, bypassKey, audience,
  verifyIdToken, finalize })`. Steps: empty-key defensive 500;
  header present + lowercase hex regex; `timingSafeEqual` against
  `HMAC-SHA256(key, bodyText)`; JSON-parse body; require

## Design notes

- **Hex over base64url for the header.** Trivial to produce with
  `crypto.createHmac().digest('hex')` in shell/test helpers; no
  padding awkwardness; doesn't pull `b64uDecode` out of
  `packages/auth`'s internals.
- **Bypass key is opaque bytes.** Treat the env value as UTF-8
  bytes; no decoding ceremony in the route. The shared-secret-ness
  is what matters, not the key's representation.
- **JWKS verify still runs.** This is the entire reason for the
  route: iter 129's production-down was a `jose` module-not-found
  on the JWKS path, which a code-exchange-skipping shortcut would
  re-introduce.
- **404 when key unset.** Cheaper than 401 + Constant-time check
  on every probe, and means non-test deploys never expose the
  route at all. The Fly secret in `live` is what flips it on.
- **Empty key still hits 500.** Defensive: if `TEST_OAUTH_BYPASS_KEY=`
  (empty) is somehow set in env, the route reaches the
  orchestrator, which 500s rather than accepting any signature
  over an empty key.
