## Approach

- New `tests_gold/lib/test/supertexDaemonReal.test.mjs`. Skips
  with exit 0 when either `vendor/supertex/build/supertex` is
  missing or `which lualatex` fails — supertex builds are owned
  by that repo / the sidecar Dockerfile, so a fresh checkout
  without the submodule built should still pass the suite.
- Dynamic-import the `SupertexDaemonCompiler` source by absolute
  path after the skip-gates, mirroring how `tests_gold/lib/test/
  cleanupProjectMachine.test.mjs` reaches into `apps/web` source.
- Write a small 2-page LaTeX fixture, spawn the compiler with
  generous `readyTimeoutMs`/`roundTimeoutMs` (60s/120s) since
  cold lualatex initialisation is slow on first run, run two
  compiles, assert each segment starts with `%PDF` and is > 1 KB.
  Second compile exercises the persistent-process invariant.
- Python wrapper `tests_gold/cases/test_supertex_daemon_real.py`
  mirrors `test_local_db.py` (same `pnpm exec tsx` shape, 300s
  timeout).
