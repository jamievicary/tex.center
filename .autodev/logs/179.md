## Approach

`createToastStore({ now, setTimeout, clearTimeout })` exposes
the `subscribe`/`push`/`dismiss`/`clear` Svelte-store contract.
Defaults `now` → `Date.now`, `setTimeout` → global. Tests
inject a manual clock so timer-based assertions are
deterministic — no real `setTimeout`/sleeping in the unit
suite. Module-level singleton `toasts` is what the app
consumes; tests instantiate their own via `createToastStore`.

Aggregation semantics: every push checks `lastPushAt.get(key)`;
if found within `AGGREGATE_WINDOW_MS` (500ms), the existing
toast's `count` bumps, `text` updates to the latest push's
text, the TTL re-arms (so a stream of merges extends the toast
visibility), and we return the existing id. A push past the
window — or with no matching key, or a missing key — produces
a fresh toast.

Per-category default TTLs encoded in `DEFAULT_TTL_MS`. Explicit
`ttlMs` overrides. `persistent: true` disables auto-dismiss
entirely; the consumer's dismiss button (rendered as `×`)
invokes `dismiss(id)`.

`Toasts.svelte` is a thin renderer: fixed bottom-right stack,
color from category, `data-toast-category` / `data-toast-id` /
`data-toast-count` attributes so future Playwright specs can
target without text matching. `pointer-events: none` on the
stack with `pointer-events: auto` on each toast so the toasts
don't block the editor below them.

## Notes for future iterations

- Next M9.editor-ux slice: wire `file-op-error` (red, dedup by
  reason) and compile errors (error category, dedup by detail)
  as the first user-facing consumers — small, testable via
  GT-E. Save-feedback affordance can follow.
- Debug-mode toggle: simplest wiring is a `?debug=1` query
  param that writes `localStorage.debug="1"`, then a module
  initializer reads localStorage and sets a `debugEnabled`
  flag the WS client checks before pushing debug toasts. The
  fan-out hook fits naturally as a `WsClient.onChange`
  subscriber for incoming frames + a small wrapper around
  outgoing send for the green Yjs-op category.
- The `text` field of an aggregated toast updates to the most
  recent push's text by design — `174_answer.md` wants a
  monotonic display like "Yjs op (×N)" rather than the
  original push's text frozen with a count badge. Consumers
  building debug toasts should construct text that reads well
  at count 1 and count N (e.g. `"Yjs op"` rather than
  `"Yjs op fired"`).
- The store treats no-aggregateKey pushes as never-mergeable,
  which means an unbounded stream of identical errors with no
  key would stack. User-facing consumers should pass an
  `aggregateKey` (e.g. `file-op-error:<reason>`) when they
  expect repetition.
