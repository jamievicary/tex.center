## Goal for this iteration

Land the user-facing toast-consumers slice of M9.editor-ux: wire
`file-op-error` and compile-error protocol frames to red error
toasts, with dedup by reason/detail via the existing
`aggregateKey` aggregation. Save-success toast is intentionally
out of scope — there is no server-side save-ack signal today, and
the PLAN scopes that to the separate SyncStatus slice (Yjs
provider sync state acked by sidecar persistence).

## Approach

- Extend `WsClient` with two narrow event callbacks
  (`onFileOpError`, `onCompileError`) fired on the corresponding
  decoded frames. Keeping these distinct from the broader
  `onChange` snapshot stream avoids edge-detection ambiguity (two
  identical successive `file-op-error` frames must each fire so
  the toast store can dedup via `aggregateKey`; relying on
  snapshot transitions would swallow the second one).
- `+page.svelte` wires both callbacks to `toasts.push(...)` with
  `category: "error"` and aggregateKeys
  `file-op-error:${reason}` and `compile-error:${detail}`.
- New unit test `wsClientToastEvents.test.mjs` asserts the
  callbacks fire on the right frames with the right payloads, do
  not fire on unrelated frames, and survive a missing
  `detail` on compile-status error.

## Notes for future iterations

- The next M9 slice is the debug-mode toggle + protocol fan-out
  (174_answer §Color categories). Wire point is the same — add
  `onPdfSegment`, `onCompileStatus`, `onFileList`, `onHello`,
  outgoing-Yjs-op hook — gated on a `?debug=1` /
  `localStorage.debug==="1"` runtime flag with a Ctrl+Shift+D
  toggle. The aggregateKey discipline applied here (per-reason /
  per-detail) carries over for the debug-red `file-op-error`
  case, but blue `pdf-segment` and green Yjs-op want a single
  shared aggregateKey to coalesce bursts into one `×N` toast.
- GT-E (local Playwright) for the user-facing toasts is still
  pending. It needs a way to provoke a `file-op-error` from the
  fixture-backed local sidecar (e.g. create a duplicate name)
  and observe the `data-toast-category="error"` element appear.
  Sidecar's `serverCreateFile` test fixture already exercises
  the duplicate-name reject path, so a Playwright spec can
  reuse the same shape via the UI.
- Save-success toast is bundled with the SyncStatus slice (Yjs
  provider sync state acked by sidecar persistence) — the wire
  doesn't carry a persistence-ack signal yet, and adding one is
  an architectural slice in its own right.
