## Goal for this iteration

M9.resource-hygiene first slice (per PLAN.md and 173b_answer.md):
add `cleanupProjectMachine` to the `afterEach` of the five live
specs that spawn per-project Fly Machines, factor a shared
helper, and add a Machine-count guardrail gold case.

## Actions

- New shared helper:
  `tests_gold/playwright/fixtures/cleanupLiveProjectMachine.ts`.
  Wraps the existing `cleanupProjectMachine` primitive plus a
  `MachineDestroyer`/`AssignmentStore` pair driven by
  `FLY_API_TOKEN` + `SIDECAR_APP_NAME` env (default app name
  `tex-center-sidecar`). No-op if `FLY_API_TOKEN` is absent
  (covers non-live targets and non-maintainer environments).
  Best-effort `projects` row delete after the destroy completes.
  Catches and logs but does not rethrow — afterEach must not
  mask the body failure.
- Switched five specs from `db.delete(projects)` to
  `cleanupLiveProjectMachine`:
  `verifyLiveFullPipeline.spec.ts`,
  `verifyLiveEditTriggersFreshPdf.spec.ts`,
  `verifyLiveInitialPdfSeeded.spec.ts`,
  `verifyLiveNoFlashLoad.spec.ts`,
  `verifyLiveSustainedTyping.spec.ts`. Removed now-unused
  `eq` / `projects` imports.
- Rewrote the comment block in `verifyLiveFullPipeline.spec.ts`
  to flip from "intentionally NOT called" to the
  correctness-vs-optimisation framing from 173b_answer.md.
- Reused-pipeline (`verifyLiveFullPipelineReused.spec.ts`)
  and WS-upgrade (`verifyLiveWsUpgrade.spec.ts`) deliberately
  untouched. The former's premise requires Machine retention;
  the latter already wires its own destroyer/store (kept
  separate to minimise iteration blast radius).
- New gold case
  `tests_gold/cases/test_sidecar_machine_count.py`. Pulls
  `creds/fly.token`, GETs `api.machines.dev/v1/apps/<app>/machines`,
  asserts count ≤ `TEXCENTER_MAX_SIDECAR_MACHINES` (default 5).
  On breach: lists id/name/state/created_at per Machine. Skips
  silently when `creds/fly.token` is absent.
