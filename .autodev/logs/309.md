## Goal for this iteration

Land **M22.4a** (the UI-only half of the toast-UX + compile-cycle
telemetry batch queued in `306_answer.md`). No wire change. Concretely:

- `EditorSettings.debugMode: boolean` (default `true`) joins `fadeMs`.
- `FADE_MS_DEFAULT` 180 → 1000 ms (default cross-fade per (3)).
- Debug-mode resolution gains a one-shot migration from
  `localStorage["debug"]` → `editor-settings.debugMode` and unifies
  URL / Ctrl+Shift+D / settings-popover writes onto the same key.
- All `debug-*` TTLs 2 s (red 4 s) → 10 s; info/success/error unchanged.
- `Toasts.svelte` keyed each gains `animate:flip` (500 ms cubic-out)
  so existing toasts visibly descend when a new one is pushed.
- New `compileCycleTracker.ts` (pure, injectable clock) prefixes
  `compile-status idle`/`error` toasts with `${elapsed.toFixed(1)}s — `.
  Editor wires it into `onDebugEvent` so cycle state stays in sync
  even while toasts are muted.

`pdf-segment` toasts still show `${bytes}B` only; bytes+page-name lands
in M22.4b along with the wire-format bump (header 13 → 17 bytes).

## Approach

- **Settings store rewrite.** Added `debugMode` to `EditorSettings`,
  `DEBUG_MODE_DEFAULT = true`, a `coerceDebugMode` guard mirroring
  `clampFadeMs` (boolean → boolean; everything else → default).
  Bumped `FADE_MS_DEFAULT` 180 → 1000. Updated parse/serialize to
  round-trip both fields and drop unknown keys.
- **Debug-mode resolution rewrite.** Renamed `initDebugFlag` →
  `initDebugMode(params, storage, settingsDebug) →
  { debug, shouldPersist }`. URL overrides; legacy
  `localStorage["debug"]` migrates and is removed on first read.
  Caller (the editor onMount) persists when `shouldPersist=true`.
  `onDebugKeyShortcut` left unchanged — caller's setter now writes
  into settings rather than the legacy key.
- **Toast store TTLs.** Promoted `DEFAULT_TTL_MS` to a named export
  (so the lock test can assert the table) and bumped all five
  `debug-*` rows to 10 000 ms. `info`/`success`/`error` unchanged
  to preserve M22.3's ergonomics.
- **`Toasts.svelte`.** One-line additions: `import { flip } from
  "svelte/animate"`, `import { cubicOut } from "svelte/easing"`,
  and `animate:flip={{ duration: 500, easing: cubicOut }}` on the
  keyed each block.
- **`compileCycleTracker.ts`.** Pure module wrapping
  `debugEventToToast`. `running` events reset `cycleStart = now()`;
  `idle`/`error` consume it and prefix the toast text. Other event
  kinds pass through unchanged. Editor wires `compileCycle.observe`
  unconditionally so cycle bookkeeping survives debug-mode toggles
  (otherwise toggling off mid-cycle would orphan the start).
- **Editor +page.svelte.** Removed `debugEventToToast` import (the
  tracker holds it now). Migrated debug-mode source-of-truth from
  the local `debug = $state(false)` rune to `settings.debugMode`;
  Ctrl+Shift+D + URL flow + popover checkbox all converge through
  `setDebugMode(next)` which writes to `editor-settings`. Settings
  popover gained a `<label class="settings-row">` debug-mode
  checkbox above the existing slider; the slider keeps its
  bind:this so M19.3's open-focus contract survives. onMount now
  parses settings first, then runs `initDebugMode` against the
  legacy key + URL, applies the resolution to `settings`, and
  persists when needed.
