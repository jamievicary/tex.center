## Goal for this iteration

Fix the `editor.spec.ts` regression visible in iter-203's gold run
(both `[local]` and `[live]` FAIL on the topbar
`getByRole("img", { name: "tex.center" })` assertion). The editor
route's brand wrapper was a clickable `<a aria-label="tex.center">`
with no `role="img"`, so the test's role-based lookup found
nothing. Projects route already uses `<span role="img"
aria-label="tex.center">` (per PLAN); editor needs the same
accessible role while keeping the anchor clickable (clickable-logo
feature is M9.editor-ux locked).

## Approach

Wrap the inlined SVG inside the anchor in a
`<span role="img" aria-label="tex.center">`. The anchor stays a
link (so clicking it still navigates to `/projects`), and the
inner span exposes `role="img"` with the accessible name the spec
queries. CSS rule `.brand :global(svg)` matches descendants so
sizing is unaffected.

## Actions

- Edited `apps/web/src/routes/editor/[projectId]/+page.svelte:112`:
  changed `<a … aria-label="tex.center">{@html linearLogo}</a>` to
  `<a … class="brand"><span role="img" aria-label="tex.center">{@html linearLogo}</span></a>`.
