## Goal for this iteration

Land the debug-mode protocol fan-out slice of M9.editor-ux per
`174_answer.md`. WsClient gains a single `onDebugEvent` typed
hook covering all six observable events (incoming pdf-segment /
compile-status / file-list / hello / file-op-error and outgoing
doc-update). The editor page gates a debug subscription on a
`debug` flag derived from URL `?debug=1`, `localStorage.debug ===
"1"`, or a Ctrl+Shift+D toggle; when on, events fan out to the
toast store as `debug-blue|green|grey|orange|red` toasts with
aggregateKeys that coalesce bursts into `×N` badges.

User-facing red toasts for file-op-error and compile-error
continue to fire unconditionally (iter 186 wiring untouched).
The duplication with debug-red on file-op-error is intentional
and acceptable — debug mode is a power-user diagnostic, not a
production path.

GT-F (local Playwright debug-toast spec) is a separate slice;
this iteration ships the WsClient hook, the page toggle, and a
node unit test for the hook.

## Actions

- `apps/web/src/lib/wsClient.ts`: added `WsDebugEvent` typed
  union and `WsClientOptions.onDebugEvent`. Fan-out points:
  pdf-segment (bytes), compile-status (state + optional detail —
  emitted as a discriminated value to satisfy
  `exactOptionalPropertyTypes`), file-list (count), file-op-error
  (reason), hello (protocol — also handled for the first time;
  previously fell through the control switch silently). Outgoing
  Yjs ops fire on a successful socket write only; `send()` now
  returns `boolean` so the doc-update path can gate the event on
  whether the frame actually reached the wire. Pre-open
  `text.insert` no longer reports as outgoing.
- `apps/web/src/lib/debugToasts.ts`: new module mapping
  `WsDebugEvent` → `ToastInput`, plus `initDebugFlag(params,
  storage)` and `onDebugKeyShortcut(target, get, set)` for the
  `?debug=1` URL precedence + localStorage persistence + Ctrl+
  Shift+D toggle. Single-key aggregation for pdf-segment / yjs-
  op (burst coalescing); per-state for compile-status; per-
  reason for file-op-error.
- `apps/web/src/routes/editor/[projectId]/+page.svelte`: imports
  the helpers; `onMount` seeds `debug` from URL/storage and
  installs the keyboard handler; supplies `onDebugEvent` that
  no-ops when `!debug` and pushes a debug toast when on. The
  `onFileOpError` / `onCompileError` user-facing red toasts from
  iter 186 are untouched; in debug mode the red user toast and a
  debug-red toast both fire (different TTLs; documented).
- `apps/web/test/wsClientDebugEvents.test.mjs`: covers all six
  event kinds, the outgoing-doc-update pre-open suppression, the
  doc-update-from-server origin filter, and `debugEventToToast`
  category + aggregateKey mapping (including keyed-by-state
  compile-status vs single-key pdf-segment).
- `apps/web/test/debugToastsToggle.test.mjs`: covers URL
  precedence (`?debug=1`/`=0`/`=foo`), storage fallback,
  Ctrl+Shift+D toggling, modifier requirements, key case
  (D/d), preventDefault on a real toggle, and listener
  detachment.
- `tests_normal/cases/test_node_suites.py`: registered the two
  new node cases.

## Notes for future iterations

- Next M9 slice: GT-F (local Playwright). Provoke a single
  keystroke, assert a `data-toast-category="debug-green"` toast
  appears; rapid typing aggregates (badge `×N`); without
  `?debug=1`, no debug toasts appear. The toast component
  already renders the category as a `data-toast-category` attr
  per iter 179 — verify when writing the spec.
- The `onDebugEvent` is wired but does NOT fire `outgoing-doc-
  update` for the *initial* state sent at WS open (which goes
  through Y.encodeStateAsUpdate at sync time, not the
  `doc.on("update")` handler used here). If GT-F needs to see
  the initial-sync outgoing toast, the y-websocket-style sync
  handshake would need its own debug event — out of scope for
  this slice.
- The current FakeWebSocket fakes used in `wsClient*.test.mjs`
  needed `instance.OPEN = 1` (in addition to the static) once
  the outgoing send path got exercised. Future tests that drive
  outgoing frames should remember this.
- Save-success / SyncStatus slice still needs the sidecar
  persistence-ack signal — unblocked only by adding that wire
  message.
