## Problems encountered

- Initial ssh-probe with `bash -lc` returned "supertex: no engine
  found" — login-shell PATH didn't include `/opt/engine/bin`.
  Re-running with `bash -c` (non-login) used the container's
  `PATH=/pnpm:/opt/engine/bin:…` and the probe worked. Not a code
  bug; a probe-script gotcha worth noting for future deploy
  verifications.
- First heredoc-driven probe through `flyctl ssh console -C …`
  produced shipout chunks but no `main.pdf`. Switching to
  `printf %s … > main.tex` (so supertex's stdin isn't the heredoc)
  fixed it; the resulting `main.pdf` is a valid 56 KB PDF.
  Suggests `supertex --once` reads stdin in some mode and an
  open-ended heredoc-as-stdin interferes; documenting here so
  M7.1.3.2's probe doesn't trip on the same thing.
