## Goal for this iteration

`.tmp.<pid>.<ms>` files have been tracked in
`apps/web/src/routes/editor/` since iter 10:

- `+page.svelte.tmp.27463.1778435906772`
- `+page.svelte.tmp.27463.1778435910622`

Edit tool — both are stale copies of `+page.svelte`. They've been
clutter for ~40 iterations. The root `.gitignore` (added iter 25
with a comment naming exactly this incident) already excludes the
pattern, but doesn't help retroactively for already-tracked files.

## Approach

- `git rm` the two files.
- Add a structural check that fails if any tracked path matches
  `*.tmp` or `*.tmp.<digits>(.<digits>)*` — the gitignore patterns
  promoted to an assertion. If somebody bypasses the gitignore
  (e.g. `git add -f`, or adds a rule-violating file before the
  rule existed), CI catches it instead of trusting the rule alone.
