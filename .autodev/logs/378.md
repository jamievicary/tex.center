# Iteration 378 — M27 dirty-page wire-through + UI overlay

- **Start commit:** `f6d0317`
- **Start time (UTC):** 2026-05-17T20:00:47Z
- **Driven by:** human conversation (not autodev).

## Goal

Pull the M27 `supertex --daemon` cutover from upstream (vendor
submodule jump `aaa625a` → `d58ddce`) and surface its new
`[dirty D]` stdout line end-to-end:

  - Sidecar parses `[dirty D]`, drops `[rollback K]` (removed
    upstream), forwards `D` on every CompileResult.
  - Wire: new `{ type: "dirty-page", page }` control message,
    broadcast AFTER the round's PDF segments so the FE can
    combine `D` with the just-shipped `shipoutPage`.
  - Web client: snapshot grows `dirtyFromPage: number | null`
    with the combine rule `candidate = max(D, highestShipout
    + 1)`, advanced forward whenever a wider segment lands.
  - PdfViewer: pages with `data-page >= dirtyFromPage` get a
    translucent grey wash + CSS-only spinning ring. On scroll
    into a dirty page, the existing `maxVisible`
    IntersectionObserver path bumps `viewing-page`, the
    sidecar coalescer kicks `recompile,N`, new chunks land,
    frontier advances, overlay clears.

## Why now

Upstream M27 reshapes the engine-pause story (paused at
INITIAL_INPUT, `recompile,T` becomes the only advance verb,
edits announce themselves on stdout instead of being inferred
from a deleted snapshot file). The pre-M27 `[rollback K]`
event has no successor in the daemon stdout grammar; leaving
the sidecar pointed at the old shape would silently fail every
post-edit round at the parser. The user-visible payoff is the
overlay: pre-M27 the FE had no signal that "the page you're
looking at is stale" — an edit that didn't change page-2
content but invalidated page-2's typesetter state still let
the FE display the unchanged-bytes render with no hint that
the engine was about to reship it.

## Files touched

### Sidecar parser + compiler

- `apps/sidecar/src/compiler/daemonProtocol.ts` — `DaemonEvent`
  drops `{ kind: "rollback"; k }` and gains `{ kind: "dirty";
  d }`. Regex/parse paths swap accordingly. Doc-comment
  rewritten for the M27 grammar.
- `apps/sidecar/src/compiler/supertexDaemon.ts` —
  `collectRound` tracks `dirtyPage: number | null`, stamps it
  on the returned `CompileResult` (success path AND the
  `maxShipout < 0` no-op path). `daemon-round-done` structured
  log carries `dirtyPage` when present. `describeEvent`
  helper updated.
- `apps/sidecar/src/compiler/types.ts` — `CompileSuccess`
  grows optional `dirtyPage?: number` with the combine-rule
  doc comment.

### Sidecar server

- `apps/sidecar/src/server.ts` `runCompile` — after the
  per-segment broadcast loop, if `result.dirtyPage` is a
  number, `broadcast(p, encodeControl({ type: "dirty-page",
  page: result.dirtyPage }))`. Ordering is load-bearing: the
  segment must land first so the FE knows the highest fresh
  shipout when the dirty marker arrives.

### Wire protocol

- `packages/protocol/src/index.ts` — `ControlMessage` union
  gains `{ type: "dirty-page"; page: number }`. No new tag
  byte (reuses `TAG_CONTROL`); existing encoder/decoder paths
  cover it via JSON.

### Web client + UI

- `apps/web/src/lib/wsClient.ts` — `WsClientSnapshot` gains
  `dirtyFromPage: number | null`. Two new private fields:
  `_highestShipout` (running max of observed `shipoutPage`)
  and `_dirtyFromPage`. Combine rule lives in two places:
  the `pdf-segment` arm (advance frontier past `shipoutPage`
  when it covers the dirty range) and the new `dirty-page`
  control arm (`candidate = max(D, _highestShipout + 1)`;
  `min` with any prior frontier).
- `apps/web/src/lib/PdfViewer.svelte` — new
  `dirtyFromPage?: number | null` prop. `syncDirty()` toggles
  `.pdf-page-dirty` class on every `[data-page]` wrapper;
  called after each render commit AND in a reactive
  `$effect` so the boundary moves without waiting for the
  next render. CSS adds `::before` (rgba(220,220,220,0.45)
  wash) and `::after` (36×36 CSS-only spinning ring with
  `pdf-page-dirty-spin` keyframes). Stacks above the existing
  absolutely-positioned canvas children via z-index 2/3.
- `apps/web/src/routes/editor/[projectId]/+page.svelte` —
  threads `snapshot.dirtyFromPage` through to PdfViewer; adds
  the new field to the initial snapshot literal.
