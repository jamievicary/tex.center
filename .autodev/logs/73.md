## Goal for this iteration

M6.3.1 ‚Äî eight-step live deploy of the control plane to
`https://tex.center`, running `flyctl`, `gh`, and the Cloudflare API
against live services using `creds/{fly,github,cloudflare}.token`.

## Actions

1. **Toolchain bootstrap.** Neither `flyctl` nor `gh` was on PATH;
   no `curl` either. Used `wget` to fetch
   `flyctl_0.4.49_Linux_x86_64.tar.gz` and `gh_2.63.2_linux_amd64.tar.gz`
   into `~/.local/bin/`. (Latest-release shortlink on flyctl repo
   404'd; resolved real tag via the GitHub releases API.)
2. `flyctl apps create tex-center` ‚Äî created in `fra` under
   personal org "Jamie Vicary" (trial plan).
3. `gh secret set FLY_API_TOKEN < creds/fly.token` on
   `jamievicary/tex.center`.
4. `flyctl deploy --remote-only --app tex-center` ‚Äî first attempt
   built fine but the new VM crash-looped to its 10-restart cap
   with `SyntaxError: Cannot use import statement outside a module`
   on `/app/build/index.js:1`. **Root cause:** adapter-node emits
   ESM, but the runtime stage of `apps/web/Dockerfile` only carried
   `build/` ‚Äî no sibling `package.json` with `"type":"module"`, so
   Node treated `.js` as CJS. **Fix:** added a one-line stub ‚Äî
   `RUN echo '{"type":"module"}' > package.json` ‚Äî to the runtime
   stage before the existing `EXPOSE 3000` line. Redeploy succeeded;
   machine reached good state.
5. `flyctl ips allocate-v4 --yes` ‚Äî **trial-org block**: "This
   functionality is disabled for trial organizations." Fly already
   gave the app a shared IPv4 `66.241.125.118` plus a dedicated
   IPv6 `2a09:8280:1::114:4adc:0` at first deploy; SNI on the
   shared v4 routes correctly for custom apex domains, so the
   apex `A` record points at the shared address. Dedicated v4
   needs a paid org and is logged in `deploy/README.md`.
6. `flyctl certs create tex.center --app tex-center` ‚Äî created in
   "Awaiting configuration" state.
7. Cloudflare DNS via `scripts/cloudflare-dns.mjs`. The script's
   `--token-file` expects a raw bearer token; `creds/cloudflare.token`
   is a JSON object (`{ token, zone_id, zone }`). Extracted `.token`
   to a temp file, ran the script with `--ipv4 66.241.125.118
   --ipv6 2a09:8280:1::114:4adc:0`; one A update + one AAAA create.
   No ACME challenge record needed ‚Äî Fly fell back to TLS-ALPN-01
   once apex resolved, cert issued ~30s later (RSA + ECDSA dual
   chain from Let's Encrypt).
8. Probed via `wget -S`: `https://tex.center/healthz` ‚Üí 200
   `{"ok":true,"protocol":"tex-center-web-v1"}`;
   `https://tex.center/` ‚Üí 200 HTML.
9. Wrote `deploy/README.md` capturing app id, region, IPs, DNS
   records, cert state, and the JSON-token extraction recipe.
