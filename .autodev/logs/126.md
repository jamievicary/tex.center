## Goal for this iteration

Pick up the FUTURE_IDEAS bullet "Cloudflare token-file JSON
support". `scripts/cloudflare-dns.mjs --token-file` previously
expected a raw bearer string, but `creds/cloudflare.token` is
actually `{ token, zone_id, zone }` JSON — iter 73 worked around
this by extracting `.token` to a temp file before invoking the
reconciler. Teach the script to auto-decode JSON when the file
body starts with `{`, so the credential file works directly.

## Approach

- Add a small pure helper `parseTokenFile(body)` exported from the
  script: trim, treat a leading `{` as JSON and require a non-empty
  `.token` string, otherwise treat as a raw bearer. Explicit errors
  for empty body, malformed JSON, and missing/empty `.token` field.
- Rewire `main()` to call `parseTokenFile` and prefix any thrown
  error with the file path for the operator.
- Cover the helper in `scripts/test/cloudflareDns.test.mjs` — every
  branch including whitespace tolerance and the malformed-JSON
  path. The CLI itself remains untested (composition only).
