## Goal for this iteration

Open M6 (Fly deploy of the control plane) with its first coherent
slice: a multi-stage `apps/web/Dockerfile` plus matching
`.dockerignore`, and a static structural test that prevents
silent drift between the install layer and the pnpm workspace.
M5 closed at iter 39, M3 was re-scoped at iter 41, and M6 is the
remaining structural milestone before acceptance. `fly.toml` and
the GitHub Actions workflow are deliberately deferred to M6.1/M6.2
— they're independently testable and benefit from the Dockerfile
being settled first.

## Approach

- Adapter-node bundles every third-party dep into
  `apps/web/build/`, so the runtime stage needs only that
  directory plus the Node base image. No `node_modules` shipped,
  no pnpm in the runtime layer. Confirmed by grepping the build
  output for non-`node:` non-relative imports — only `./shims.js`.
- Builder stage installs the **whole** workspace (manifests-only
  COPY first, then `pnpm install --frozen-lockfile`, then source)
  so the install layer caches across source edits. The build is
  scoped with `pnpm --filter @tex-center/web build`.
- We can't actually build the image inside `tests_normal` (no
  docker, and the pull would dwarf the iteration budget). The
  cheapest test that catches the most likely regression is
  manifest-coverage: a new `packages/foo` added to
  `pnpm-workspace.yaml` without a matching `COPY foo/package.json
  foo/` in the install layer would only fail inside the Docker
  build, far from the iteration that introduced the package. The
  static test reads `pnpm-workspace.yaml` and asserts every
  resulting package directory has a manifest COPY in the
  Dockerfile.
