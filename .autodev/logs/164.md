## Diagnosis

Iter-163 deploy (run `25737945622`) just completed: M8.pw.4 still
red on `pdf-segment` timeout. The new `ws_proxy` JSON line in
`tex-center` logs immediately surfaced the cause:

```
2026-05-12T13:38:44Z ws_proxy.kind=resolve-error projectId=d4664912-...
  message="Fly Machines API 408 .../wait?state=started&timeout=60:
           deadline_exceeded: machine failed to reach desired state,
           started, currently created"
```

Cross-referenced with `tex-center-sidecar` logs:

- Machine `28692e1fe09d48`: image pull started 13:37:50Z, server
  listening 13:39:05Z (**1m12s after pull start**).
- Machine `7813e27c437218`: pull 13:37:31Z, listening 13:39:12Z
  (**1m38s**).

`flyMachines.waitForState` calls `/wait?...&timeout=60`; Fly caps
the parameter at 60. `driveToStarted` therefore 408'd before
either Machine finished pulling the image. The 502 the spec saw
came from the resolver's error propagating back through the proxy.

## Changes

- `apps/web/src/lib/server/upstreamResolver.ts`: new
  `coldStartTimeoutSec?: number` option (default 300). Internal
  `waitForStartedWithRetry(machineId)` loops `waitForState(..., "started",
  { timeoutSec: waitTimeoutSec })`, catching errors with
  `.status === 408` and retrying until the overall deadline.
  Replaced all three `waitForState(..., "started", ...)` callsites
  in `driveToStarted` with the helper. `stopping/suspending →
  stopped` and `startMachine` callsites keep the single-call wait;
  only the post-start "reach started" wait needs the cold-start
  budget.
- `apps/web/src/lib/server/upstreamFromEnv.ts`: passes
  `coldStartTimeoutSec: 300` so prod gets the 5-min budget. Local
  dev / fallback static-sidecar path is unaffected.
- `apps/web/test/upstreamResolver.test.mjs`:
  - Case 8: a `MachinesClient` stub that throws `{ status: 408 }`
    on the first `waitForState("started", ...)` then succeeds; the
    resolver retries once and returns the upstream.
  - Case 9: `coldStartTimeoutSec: 0` makes 408 propagate
    immediately, locking the deadline behaviour.
