## Approach

1. Add `hydrated: boolean` to `WsClientSnapshot`. Flip false‚Üítrue
   on the first `doc-update` (the Yjs initial-sync frame) or
   `file-list` control. Other control kinds (e.g. an early
   `compile-status`) must not be treated as hydration signals.
2. In the editor page, render `<Editor>` only when
   `snapshot.hydrated` is true. Replace the prior
   `placeholderDoc` indirection (a transient Y.Doc whose Y.Text
   was bound to CodeMirror until the real doc landed) with a
   same-dimensioned `.editor-placeholder` div so the grid cell
   doesn't reflow when CodeMirror appears.
3. Convert the topbar `.brand` from `<div>` to `<a
   href="/projects">` with `color: inherit` + hover underline,
   so the click navigates without changing visuals.
4. Cover the hydration flag with a Node unit test that fakes the
   browser WebSocket via `EventTarget` and dispatches real
   protocol frames at the WsClient.
