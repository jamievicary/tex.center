## Goal for this iteration

M8.pw.3.2 — land the refresh-token plumbing that M8.pw.3.3's live
spec will consume. Two pieces: a `tests_gold/lib/` helper that
trades a stored Google refresh token for a fresh ID token via the
OAuth2 token endpoint, and a one-shot CLI script that walks an
operator through obtaining that refresh token against a separate
test OAuth client.

## Actions

- `tests_gold/lib/src/mintGoogleIdToken.ts`: pure wrapper around
  the refresh-token grant. Injectable `fetchFn` + `tokenUrl`,
  default `https://oauth2.googleapis.com/token`. POSTs
  `grant_type=refresh_token`, `client_id`, `client_secret`,
  `refresh_token`. Throws on non-2xx (status + truncated body) or
  on a 2xx with missing/empty `id_token`. Rejects empty inputs
  before hitting the network.
- `tests_gold/lib/test/mintGoogleIdToken.test.mjs`: stub-fetch
  unit test. Cases: happy (asserts URL, method, form body, headers,
  `id_token` returned); `tokenUrl` override; non-2xx → throws with
  status + body; 2xx with no `id_token` → throws; empty-string
  `clientId`/`clientSecret`/`refreshToken` rejected with zero
  fetch calls.
- `tests_gold/cases/test_mint_google_id_token.py`: gold case
  wrapper (mirrors `test_mint_session.py`).
- `scripts/google-refresh-token.mjs`: one-shot CLI. Reads creds
  (default `creds/google-oauth-test.json` — the separate test
  client, *not* the production `creds/google-oauth.json`),
  generates PKCE, binds a loopback HTTP server on `--port`
  (default 4567), prints the authorize URL with
  `access_type=offline&prompt=consent`, awaits the redirect at
  `/oauth-callback`, validates the returned `state` with
  `timingSafeEqual`, exchanges the code with the PKCE verifier,
  writes the refresh token at mode 0600 to
  `creds/google-refresh-token.txt`. Errors out (with revoke-and-
  retry hint) if Google didn't return a `refresh_token` — that's
  what happens when the same OAuth client has been previously
  consented without `prompt=consent`.
- `.autodev/PLAN.md`: ticked M8.pw.3.2 with concrete wire-format
  details; bumped the "Next ordinary iteration" pointer to M8.pw.3.3.

## Design notes

- **Why a separate test OAuth client.** A refresh token is a
  long-lived credential that bypasses the consent screen. Issuing
  one against the production client would mean a leaked
  `creds/google-refresh-token.txt` could mint Google sessions
  bound to the real `client_id`. The test client is purely a
  CI/dev artefact scoped to obtaining ID tokens for the dev
  account during deploy-verify probes — separate blast radius.
- **Helper kept pure, script kept I/O-only.** Same precedent as
  `scripts/seed-live-user.mjs` (untested) vs the per-helper
  PGlite tests under `tests_gold/lib/`. The OAuth-redirect loop
  is unmockable in a useful way; the helper that the spec
  actually calls is what matters and is unit-tested.
- **HTTP server lifetime.** Single-request: the server `.close()`s
  and the promise resolves/rejects on the first hit to the
  redirect path. Both branches go through `server.close()` before
  resolve/reject so the script exits cleanly under both success
  and OAuth-error paths.

  it's a problem.
