## Goal for this iteration

PLAN priority #1 (M20.3) needs the GT-9 preservation spec to be a
truthful signal. Iter 333 landed the spec but the iter-333 gold
pass returned RED with:

> Error: no post-edit pdf-segment within budget — cannot guarantee
> the sentinel was persisted before the stop step

The failure is in the spec, not in the production persistence path.
The spec types `% preserve-<uuid>` **after `\end{document}`** —
exactly the "warm-doc body edit no-op" pattern that
`test_supertex_warm_doc_body_edit_noop` documents: the supertex
daemon detects the typeset output is unchanged and skips reshipout
(`{ok:true, segments:[], noopReason:…}`), so no `pdf-segment`
frame is ever emitted over the wire. The spec then times out
waiting for one.

(Aside: in production the persistence path likely DID write the
sentinel to Tigris — `persistence.maybePersist()` runs at the top
of `runCompile` before the compiler is invoked, so even a no-op
edit persists. The spec just has no wire signal to confirm that.)

Fix: place the sentinel inline at the end of the visible
`Hello, world!` line — same cursor choreography as GT-3 — so the
typeset PDF actually changes, the daemon reshipouts, a fresh
`pdf-segment` arrives, and **that arrival** is a wire-observable
proof that `runCompile` end-ran and `maybePersist` therefore wrote
the new source bytes to the blob store. Sentinel survives in the
source bytes either way, so the post-reopen `.cm-content`
substring assertion is unchanged.

## Approach

Update `verifyLiveGt9StoppedPreservesEdits.spec.ts`:
- Position cursor at end of the `Hello, world!` line via
  `ArrowUp/ArrowUp/End` (same pattern as GT-3, which is GREEN).
- Type the sentinel as ` preserve-<uuid>` (leading space for
  cleaner kerning in the rendered PDF, irrelevant for the
  `.cm-content` substring check).
- Drop the `% ` prefix and the "LaTeX ignores anything after
  `\end{document}`" rationale (the whole point of this iter is
  that strategy didn't probe the persistence path).
- Sentinel substring assertion stays identical — `.cm-content`
  `textContent` joins `.cm-line`s without separators, so
  `preserve-<uuid>` remains a contiguous substring whatever line
  it lands on.
