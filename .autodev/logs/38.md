## Actions

- `packages/db/src/sessions.ts` — add `getSessionWithUser(db,
  sid)`: `select({session, user}).from(sessions).innerJoin(users,
  eq(sessions.userId, users.id)).where(eq(sessions.id, sid)).
  limit(1)`. Returns `{session, user} | null`. Expiry is left to
  the caller so the helper can also be reused by future
  sweepers/admin tools.
- `packages/db/test/users-sessions-pglite.test.mjs` — appended
  happy-path and unknown-sid assertions for the new helper.
- `packages/auth/src/index.ts` — exported `VerifyFailure` type
  (already declared in `session.ts`); `sessionHook` needs it to
  map token-verify reasons.
- `apps/web/src/lib/server/sessionHook.ts` (new) — pure
  `resolveSessionHook`. Reads the `tc_session` cookie out of the
  raw header, calls `verifySessionToken`, validates the sid is
  uuid-shaped (cheap guard against passing garbage to a uuid
  column), calls the injected `lookupSession(sid)`, and finally
  asserts `session.expiresAt > nowSeconds*1000`. Returns
  `{session, clearCookie, reason}` where `reason` is a tagged
  union (`no-cookie`/`bad-token`/`expired-token`/`bad-sid`/
  `no-row`/`lookup-error`/`expired-row`/`ok`). Cookie is cleared
  on every "invalid" branch *except* `lookup-error` (DB outage —
  the row may still be good, so we don't sign the user out on a
  transient failure).
- `apps/web/src/lib/server/sessionConfig.ts` (new) —
  `loadSessionSigningKey()`. Subset of `oauthConfig` so hooks
  don't need `creds/google-oauth.json`. Returns `null` if the
  env var is unset (anonymous everywhere — the white page must
  keep rendering in early dev); throws only if the value is
  malformed.
- `apps/web/src/hooks.server.ts` (new) — SvelteKit `Handle`.
  Pulls the key, builds the `resolveSessionHook` input, populates
  `event.locals.session`, and short-circuits with a 302 to `/`
  when `/editor` is requested anonymously. `clearCookie` (if
  any) is appended both to the redirect headers and the normal
  `resolve` response.
- `apps/web/src/app.d.ts` — `App.Locals.session: ResolvedSession
  | null`. (Note the in-conversation log description said
  ResolvedSession; the type-only import doesn't drag any runtime
  cost.)
- `apps/web/src/routes/editor/+page.ts` (new) —
  `prerender = false; ssr = false`. Without this the layout-level
  `prerender = true` would emit a static `/editor.html` that
  bypasses `hooks.server.ts`.
- `apps/web/test/sessionHook.test.mjs` (new) — 10 branch cases:

## Decisions

- **Make `/editor` non-prerendered, not the whole site.** The
  layout default still prerenders unauth/static pages (white
  sign-in, future error pages); only the authenticated editor
  shell goes dynamic. `ssr = false` is preserved so the page is
  still client-rendered — the dynamic round-trip is purely to
  give the hook a chance to redirect. Alternative was to flip
  the layout default to `prerender = false`, which would have
  pulled every static page into runtime cost for no benefit.
- **`reason` enum on the hook output.** The orchestrator could
  return just `session | null`, but the explicit reason makes
  the unit test trivially complete (one assertion per branch)
  and gives the future logging layer a stable label.
- **DB outage = don't clear cookie.** Signing a user out because
  the DB blipped for one request would be hostile; the next
  request reverifies. The cookie itself is HMAC-signed so a
  forged value still fails at the token-verify gate before the
  lookup runs.
- **uuid format pre-check.** Passing a random string to a uuid
  column triggers a postgres error which would surface as
  `lookup-error` (cookie retained). Validating the format first
  lets us treat "this is structurally not one of our sids" as
  `bad-sid` (cookie cleared) — the right outcome since a real
  cookie we minted always carries a uuid.
- **Separate `loadSessionSigningKey` rather than reusing
  `loadOAuthConfig`.** `hooks.server.ts` runs for every request,
  including unauthenticated visits to `/`. `loadOAuthConfig`
  reads `creds/google-oauth.json` (and throws hard if missing),
  which would break the white page in any environment that
  hasn't wired creds yet. A signing-key-only loader has the
  minimal dep surface and degrades gracefully when unset.
- **Protected-prefix list, not declarative per-route guard.** A
  single `if (path.startsWith('/editor'))` in the hook is
  cheaper to reason about than scattering `+page.server.ts`
  loads. Other future protected prefixes (`/api/projects`,
  `/dashboard`) join the list with one line each.

## Notes for future iterations

- **Editor UI session consumption.** With `event.locals.session`
  populated, a `+layout.server.ts` returning
  `{user: {email, displayName}}` will let the editor render a
  signout affordance. Pair with `POST /auth/logout` that deletes
  the sessions row and emits a clear-cookie.
- **Session sweeper.** Stale rows past `expires_at` still
  accumulate. One-line `DELETE FROM sessions WHERE expires_at <
  now()` can ride on top of any periodic task once we have one.
- **`prerender = false` on protected pages is a hidden contract.**
  Adding a new protected route means remembering to also opt out
  of prerender. A `+layout.server.ts` under a protected
  subdirectory would centralise it; revisit when there's more
  than one.
- **JWKS clock-skew tolerance.** Still open from iter 36.
