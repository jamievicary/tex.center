## Goal for this iteration

Discussion mode: answer `.autodev/discussion/309_question.md`, a
three-point batch covering (1) toast TTL override to uniform 10 s,
(2) `\input{sec1}` regression in deployed editor, (3) reported
page-prefetch off-by-one. Maintenance cron (N%10==0 refactor) is
preempted by the unanswered question — next ordinary iteration
takes it.

Item 1 is mechanical enough to land alongside the answer (three
numbers + one test case + a header comment). Items 2 and 3 each
deserve their own milestone slice; landed as PLAN entries with
diagnosis, not code.

## Approach

- Read each point, mapped to source:
  - **(1) TTLs.** `toastStore.ts` `DEFAULT_TTL_MS`, lock test
    cases 2 and 8.
  - **(2) `\input`.** `apps/sidecar/src/workspace.ts` end-to-end +
    `apps/sidecar/src/server.ts` upload/create-file handlers +
    `apps/sidecar/src/compiler/supertexDaemon.ts` cwd/args.
    Root cause confirmed: `ProjectWorkspace` only exposes
    `writeMain(source)`. Auxiliary files via `addFile` /
    `upload-file` mutate Yjs + blob storage but never reach the
    on-disk workspace dir — so `\input{sec1}` cannot resolve
    through lualatex kpathsea against `cwd = workDir`. The user's
    suspect (a) is correct.
  - **(3) Page-prefetch.** `apps/sidecar/src/server.ts:528`
    hard-codes `targetPage: 0` → `recompile,end`. There is no
    target-page gate active today (commented out at iter 295-ish);
    `maxViewingPage(p)` survives only as input to
    `coalescer.kickForView`, which is a scrolled-past-emit
    trigger, not a per-compile clamp. So a `pickMaxVisible`
    off-by-one cannot be the mechanism. The actual no-segment
    behaviour for "edit on page N+2" must be inside supertex's
    own checkpoint/incremental engine deciding the round emits no
    chunks. Wrote up M21.3 to instrument and pin this rather than
    speculate.

- Decided to **not** bundle item 2 into this iteration even though
  it would close a categorical product regression. The fix spans
  `ProjectWorkspace`, `persistence.ts`, cold-boot rehydration, and
  a new gold spec — that's a coherent multi-slice milestone (M23),
  not a discussion-bundled patch. Queued at the top of the active
  priority list (above M20.2 / M21.x / M22.x perf work).
