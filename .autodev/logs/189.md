## Goal for this iteration

Land the sidecar half of the iter 188 edit→preview regression fix:
remove `SupertexDaemonCompiler.assembleSegment`'s `maxShipout < 0`
directory-scan fallback. A no-op compile (round-done with no
`[N.out]` events) is now reported as `{ ok: true, segments: [] }`
rather than re-shipping stale chunks from disk as a fresh segment.
This unmasks the upstream supertex rollback no-op for follow-up
PR work (M7.4.x): the `compiling…` toast resolves to idle without
a `pdf-segment` frame, instead of silently re-broadcasting an
unchanged PDF as if the edit had taken effect.

GT-5 will remain RED on the live deploy until the upstream supertex
fix lands; that's the intended TDD shape from 188_answer.md.

## Actions

- `apps/sidecar/src/compiler/supertexDaemon.ts`: deleted the
  `maxShipout < 0` directory-scan fallback and the
  `listChunkIndices` helper. `compile()` short-circuits to
  `{ ok: true, segments: [] }` when `events.maxShipout < 0`;
  `assembleSegment` now precondition-asserts `maxShipout >= 0`.
- `apps/sidecar/test/supertexDaemonCompiler.test.mjs`: new FAKE
  mode `noop` (round-done, no shipouts, no error) and a case
  asserting `{ ok: true, segments: [] }`. Also a `noop-stale`
  case where pre-existing `*.out` files in `chunksDir` MUST be
  ignored (the regression-shape lock for this slice).

## Notes for future iterations

- GT-5 remains RED on the live deploy by design. The next
  visible failure mode in the live editor is:
  `compile-status: running` → `compile-status: idle` with no
  `pdf-segment` frame in between. That signature is the
  diagnostic shape the M7.4.x upstream PR should reproduce
  against vendor/supertex's daemon — process_event finding no
  rollback target.
- Capturing that repro is a vendor/supertex-only change (no
  tex-center plumbing), so it sits outside the autodev loop's
  cheap test cycle. Easiest path: a tiny C-level test under
  `vendor/supertex/tests/` driving `recompile,end` against a
  freshly-typeset doc that's then edited; assert
  `--daemon` emits at least one `[N.out]` between
  `[rollback K]` and `[round-done]`. M7.4.x territory.
- If GT-5 ever needs to be green BEFORE the upstream fix lands,
  the sidecar could emit a synthetic `compile-status: error`
  detail when `maxShipout < 0`, but that is wrong: the round
  did not error — the rollback path found nothing to roll back
  to. Better to surface the real shape upstream than paper over
  it client-side.
