## Goal for this iteration

Refactor cron (`N%10==0`, deferred from iter 180 once for the
M8.pw.4 regression, once for the cold-start flake investigation
— second-and-final defer per §3, must land now).

Highest-leverage refactor in the gold-spec surface: extract the
two duplicated primitives that have grown across five live specs.

1. **`framereceived` WS listener + tag constants** — verbatim in
   `verifyLiveFullPipeline.spec.ts`,
   `verifyLiveFullPipelineReused.spec.ts`,
   `verifyLiveGt2InitialPdfSeeded.spec.ts`,
   `verifyLiveGt3EditTriggersFreshPdf.spec.ts`,
   `verifyLiveGt4SustainedTyping.spec.ts`. Each spec re-declares
   `TAG_PDF_SEGMENT = 0x20` (and GT-C/D also `TAG_CONTROL = 0x10`)
   and attaches the same listener filtering by per-project WS URL.
2. **Bounded canvas-painted poll** — iter 182 fixed the
   `.preview canvas` race in fullpipeline + reused by replacing a
   single-shot `evaluate` with a 30s `expect.poll` re-locating the
   canvas. The same single-shot pattern still lives in GT-B
   (`verifyLiveGt2InitialPdfSeeded.spec.ts`), exposed to the
   identical race. Without consolidation, future GT specs adding
   canvas assertions will copy whichever shape is nearest.

Land as two fixture modules + 5 spec rewrites, propagating the
anti-flake bounded poll to GT-B in the process. Update the
normal-suite structural checks (which currently anchor on
`TAG_PDF_SEGMENT`, `0x20`, `framereceived`, `TAG_CONTROL` tokens
inside the spec bodies) to anchor on the helper files instead,
plus helper usage in the specs.
