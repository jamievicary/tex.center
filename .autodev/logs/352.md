## Approach

Per `.autodev/discussion/345_answer.md` §B steps 1–4. Key design
choices:

- **Split into two TS modules.** `wireTimelineFormat.ts` is a pure
  data + formatter module with **no Playwright import** — kept
  unit-testable from a plain Node script (`tests_normal/cases/
  wireTimelineFormat.test.mjs`). `wireFrames.ts` is the
  Playwright-side adapter that converts `framereceived`/`framesent`
  events into `TimelineEntry[]` and threads the formatter.
- **Backward-compat preserved.** `captureFrames(page, projectId)`
  return shape is unchanged for the 5 existing consumers
  (`pdfSegmentFrames`, `overlapErrors`, `docUpdateSent`,
  `compileStatusEvents`); the new `dumpTimeline(specName)` method
  is additive.
- **Auto-binding via `captureFramesAuto(page)`.** Sniffs project
  UUIDs from the WS URL (`/\/ws\/project\/([0-9a-f-]{36})/`) so the
  fixture doesn't need a projectId argument. Wired into
  `authedPage`'s fixture body — every spec (live + local) inherits
  it. Local-target specs that never open a per-project WS hit the
  "no project WS observed" branch and emit one uniform line, keeping
  the gold transcript shape consistent.
- **Compile-cycle semantics in pure code.** A "cycle" is
  `running → idle/error`. Back-to-back `running` (sidecar coalesces
  overlapping recompiles) closes the prior cycle with whatever
  segment count it had. An unclosed final cycle is silently dropped
  — we don't yet know if it would have shipped. Tested directly.
- **Env-gated dump.** `TEXCENTER_DUMP_WIRE_TIMELINE=1` toggles the
  fixture-teardown `console.log`. Default-on in
  `tests_gold/cases/test_playwright.py` (via `env.setdefault(...)`)
  so the local iter harness gets the data; any future hosted CI
  runner can opt out by exporting `=0`.
