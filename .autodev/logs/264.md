## Goal for this iteration

M14.title-bar — first item on the post-iter-262 priority queue.
Add a centred project title to the editor topbar with a DOM-only
locking assertion, pin + fix bundled (the milestone is a small UI
slice, so a pre-RED iteration would be pure overhead).

## Approach

1. Render `data.project.name` as `<h1 class="project-title">` in
   the `.topbar` of `apps/web/src/routes/editor/[projectId]/+page.svelte`,
   between `.brand-group` (left) and the `who` / sign-out group
   (right).
2. Switch the topbar from `display: flex; justify-content:
   space-between` to `display: grid; grid-template-columns: 1fr
   auto 1fr`. The two 1fr tracks are equal-width by definition,
   so the auto centre track is mathematically centred within the
   topbar regardless of left/right content widths — robust to
   topbar resize per the M14 requirement. Add `justify-self`
   directives to anchor brand-group `start`, title `center`, who
   `end`.
3. When `data.project` is null (defensive path —
   `+page.server.ts` 404s before this in normal flow), render an
   empty `<span>` placeholder so the grid still has three
   children and the layout doesn't collapse.
4. `project-title`: 0.9 rem, weight 500, `white-space: nowrap;
   overflow: hidden; text-overflow: ellipsis; min-width: 0;
   max-width: 100%` so very long names truncate rather than push
   the side groups out of their 1fr tracks. Style alignment with
   M16 (Source Serif 4) deferred to that milestone.
5. Lock the centring in `tests_gold/playwright/editor.spec.ts` —
   the existing local-target "renders three-panel layout" case
   gets a new assertion block that fetches `boundingBox()` for
   `.topbar` and `[data-testid="project-title"]` and asserts
   `|titleCentre − topbarCentre| ≤ 2 px`. 2 px absorbs sub-pixel
   rounding without admitting a real off-centre regression.

## Notes for future iterations

- The grid `1fr auto 1fr` layout silently relies on brand-group
  and who-group each fitting inside one third of the topbar
  width. At today's content (~70 px brand + ~150 px who against
  a typical 1280 px viewport) this is comfortable. If either
  side group grows materially (a future status indicator, an
  account menu) the 1fr tracks could be squeezed and push the
  centre off; if that happens, swap to `position: absolute;
  left: 50%; transform: translateX(-50%)` on the title.
- M16.aesthetic should pick this up and apply Source Serif 4 +
  the larger 1.0 rem / ~weight 600 sizing from the palette
  proposal. The `data-testid` and class names are stable hooks
  for that change.
- The empty-`<span>` fallback for null `data.project` is a
  belt-and-braces detail: server-load already 404s on
  unauthenticated / not-owned paths, so it's never observed in
  practice — kept only to preserve grid arity if someone changes
  the load contract.
