## Goal for this iteration

Make a concrete start on M6.3 (custom domain `tex.center` via
Cloudflare) by landing the in-tree half: a one-shot DNS reconciler
script that the human (or a future workflow step) runs against
`creds/cloudflare.token` to upsert the apex A/AAAA records and the
ACME TXT challenge Fly emits. Iter 44 flagged this work as
"mostly out-of-tree, possibly defer entirely" — but the
reconcile-vs-overwrite logic, the "never touch unmanaged records"
guard, and the URL/method shape of each API call are exactly the
parts that benefit from unit tests written before someone runs
this against a production zone. Out-of-tree pieces (`flyctl
certs create`, capturing the issued IPs and ACME challenge) stay
deferred to the next out-of-tree session as M6.3.1.

## Approach

- New `scripts/cloudflare-dns.mjs` — pure `reconcileRecords` +
  `buildDesired`, plus thin I/O wrappers (`fetchZoneId`,
  `listRecords`, `createRecord`, `updateRecord`, `deleteRecord`)
  all taking an injectable `fetch` so tests run offline. CLI
  reads the token from `creds/cloudflare.token` and supports
  `--dry-run`.
- Match records by `(type, name)`; never delete a record whose
  key isn't in the desired set. Update on content/ttl/proxied
  drift. If Cloudflare somehow returns duplicates for a managed
  key, keep the first and delete the rest.
- `scripts/test/cloudflareDns.test.mjs` — `node:assert`-style,
  same style as `packages/auth/test/*`. Stub fetch via a
  `(matcher, responder)` list. Cover every reconcile branch and
  every wrapper's URL + method + auth header.
- Wire into `tests_normal/cases/test_node_suites.py` as
  `test_scripts_cloudflare_dns`.

## Notes for future iterations

- **M6.3.1 (the actual cutover) is out-of-tree.** It needs (a)
  `flyctl apps create tex-center` if not done; (b) the GitHub
  Actions workflow to run at least once so a Machine exists with
  anycast IPs; (c) `flyctl ips list -a tex-center --json` to
  capture them; (d) `flyctl certs create tex.center` to mint the
  ACME challenge Fly wants in DNS; (e) `node
  scripts/cloudflare-dns.mjs --zone tex.center --ipv4 X
  --ipv6 Y --acme-name _acme-challenge.tex.center
  --acme-value Z`. The script's `--dry-run` lets the human
  preview the ops before applying. No autodev iteration can do
  this without live tokens being exercised against live
  infrastructure; it belongs in a human session or M8 acceptance
  pass.
- **`proxied: false`** is hard-coded in `buildDesired`. We want
  Fly's edge to terminate TLS directly (the ACME cert is on Fly,
  not on Cloudflare), so Cloudflare proxying would break the
  cert handshake. If we ever switch to Cloudflare-fronted TLS we
  change this — but that's a different architectural decision.
- **No `tests_gold` coverage** of the reconcile script is needed
  — there's no scenario short of hitting real Cloudflare that
  exercises anything the unit tests don't already cover.
