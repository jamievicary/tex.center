## First-attempt dead-end (recorded for completeness)

Initial implementation made `tests_gold/lib` a workspace
package (`@tex-center/test-helpers`) to wire the
`workspace:*` deps. Structural test
`test_install_layer_covers_every_workspace_package` correctly
flagged that the `apps/web` and `apps/sidecar` Dockerfiles would
need a corresponding `COPY tests_gold/lib/package.json …` line,
and both `.dockerignore` files exclude `tests_gold` outright.
Carrying a test-only workspace package into the production
build context for the sake of a `pnpm install --frozen-lockfile`
that doesn't need it is the wrong shape — backed out and wired
the deps at the root `package.json` instead.

## Notes for future iterations

- The `local` Playwright target's biggest open design point:
  the SvelteKit dev server reads sessions through `getDb()`
  which talks Postgres-wire. PGlite is in-process to *its* Node
  and not shareable. Options for pw.1.1:
  1. `@electric-sql/pglite-socket` to expose PGlite over a TCP
     socket the dev server can connect to.
  2. Ephemeral `pg_tmp`-style Postgres in `tests_gold/setup_*`.
  3. Boot the dev server with a stub session-lookup that
     short-circuits to a hard-coded test user (mocks the DB
     dependency rather than co-locating it).
  Option 1 keeps the closest fidelity to prod and reuses the
  migrations path; option 3 is fastest to land but introduces a
  test-only code branch in `hooks.server.ts` that needs careful
  gating.
- `mintSession` accepts an injected `nowMs` so tests stay
  deterministic; the Playwright fixture should just let it
  default to `Date.now()` — the 5-minute TTL is plenty of
  headroom.
