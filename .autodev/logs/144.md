## Goal for this iteration

Land the FUTURE_IDEAS item "Unit test `loadOAuthConfig` env-vs-file
precedence". `apps/web/src/lib/server/oauthConfig.ts` was refactored
in iter 76 (env-first with dev-only file fallback, after a real
incident where a file fallback masked a prod misconfiguration), but
had no targeted unit test. M7.4.2 — the PLAN's "next ordinary
iteration" pointer — is upstream-gated (no serialise/restore wire

## Approach

The bullet predicted needing a shared temp-dir + env-stash harness
in `tests_normal/lib.sh`. In practice the test stands alone fine:
`process.env` stash/restore inline, and `GOOGLE_OAUTH_CREDS_PATH`
already lets us point the dev-file fallback at a controlled file
under `mkdtempSync`. Building a shared harness for one test would
be premature abstraction; if a second test ever wants the same
machinery, extract then.
