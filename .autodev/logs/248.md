## Findings (one-line summary)

`auto_destroy: true` + sidecar `process.exit(0)` on idle means
every idle cycle destroys the per-project Machine, so every cold
access is a full `createMachine` (image pull dominates: ~5 GB).
Smallest fix: drop `auto_destroy`, replace idle-exit with a self-
suspend API call; resume path is already correct. Three follow-up
iterations scheduled (impl, optimistic delete, new gold).
