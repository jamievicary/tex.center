## Goal for this iteration

Fix the iter-246 gold RED on `test_machine_count_under_threshold`:
6 non-shared Machines (4 tagged orphans + 2 untagged legacy) >
threshold 5. The 4 fresh tagged orphans (timestamps inside the
iter-246 gold run window) prove the iter-243 metadata tagging
shipped, but `cleanupLiveProjectMachine`'s order-of-ops still leaks
on the `destroy-fails-after-row-deleted` and
`die-between-create-and-upsert` shapes: once the assignment + project
rows are gone, nothing can look the Machine up by projectId for
retry.

The fix is to add a tag-driven sweep that runs once per gold-suite
invocation and destroys any tagged Machine whose `texcenter_project`
is no longer in `projects`. Tag- and DB-driven rather than
age-driven, so it can never destroy a real user's Machine (a
matching `projects` row keeps the Machine alive).

## Approach

1. New pure helper `tests_gold/lib/src/sweepOrphanedSidecarMachines.ts`
   that takes `listMachines`/`destroyMachine` and a
   `getKnownProjectIds(): Set<string>` source plus optional
   `protectIds` set. For each tagged Machine: if tag ∉ (known ∪
   protect), destroy with force; 404 → ok; other errors collected
   into `failed` so one bad Machine doesn't shadow the rest.
2. Unit tests at `tests_gold/lib/test/sweepOrphanedSidecarMachines.test.mjs`
   covering happy path, protect-ids, 404-as-destroyed, non-404
   collected, no-op, and empty-tag defensive case. Registered as a
   gold case via `tests_gold/cases/test_sweep_orphaned_sidecar_machines.py`
   using the same `pnpm exec tsx` launcher pattern as
   `test_cleanup_project_machine.py`.
3. New `listAllProjectIds(db)` helper in
   `packages/db/src/projects.ts` (exported via the package
   wildcard re-export); appended a coverage case to
   `packages/db/test/projects-pglite.test.mjs`.
4. Wired the sweep into the existing globalSetup teardown closure
   in `tests_gold/playwright/fixtures/liveProjectBootstrap.ts`,
   after the bootstrap's own project row is dropped: lists Fly
   Machines via REST, intersects with `listAllProjectIds`, destroys
   the orphans. A small inline `makeFlyMachineSweeper` adapter
   wraps the Fly Machines list/destroy REST calls. Sweep runs only
   when `FLY_API_TOKEN` is set, matching the rest of the teardown's
   live-gated behaviour. Failures within the sweep are logged but
   never thrown.
