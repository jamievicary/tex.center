## Goal for this iteration

Land M3.1 from `.autodev/PLAN.md`: a per-project on-disk scratch
directory mirroring the Yjs-backed `main.tex` source. The mirror
sits behind a `ProjectWorkspace` helper and is written at the head
of every compile cycle. `FixtureCompiler` continues to ignore source
content, so this is scaffolding for M3.2's `SupertexOnceCompiler` ‚Äî
the file is on disk and ready to be read when a real compiler is
plugged in. Per the multi-iteration-milestone rules, the default
codepath stays runnable and the workspace's own behaviour is fully
covered by tests.

## Approach

- New `apps/sidecar/src/workspace.ts` exporting `ProjectWorkspace`.
  Lazy `init()` (creates `<root>/<projectId>/`), atomic `writeMain`
  (write `main.tex.tmp` then rename ‚Äî supertex's M3.3 watch mode
  will inotify on this path and a half-written file would trigger
  a spurious rollback), `dispose()` removes the dir.
- Reject `projectId`s that don't match `[A-Za-z0-9_-]+` so URL path
  params can't escape the scratch root with `..`.
- `buildServer` gains an optional `scratchRoot` option. If absent
  the server `mkdtempSync`s one under `os.tmpdir()` and removes it
  on `app.close()` (the "owned" flag); externally-supplied roots
  (used by tests so they can read the mirrored file from outside)
  are left in place.
- `runCompile` writes the current `Y.Text` to `main.tex` before
  calling `Compiler.compile`. A write failure surfaces as a
  `compile-status: error` and aborts that cycle.
- `onClose` disposes every project workspace and removes the owned
  root. The fixture compiler's own `close()` path is untouched.

## Notes for future iterations

- M3.2 (`SupertexOnceCompiler`) can now read directly from
  `workspace.mainTexPath()`. The compile API still passes `source`
  as a string ‚Äî when the supertex compiler arrives, change the
  `Compiler` interface to take the workspace directory instead and
  drop the string field. That's one churn iteration deferred to
  the moment it pays for itself.
- The workspace mirror today only carries `main.tex`. Multi-file
  projects (file tree from `GOAL.md` MVP scope item 3) need
  per-file mirroring driven by the file-tree backend, which is
  still M4 work. Until then the `Y.Text` named `main.tex` is the
  whole project.
- `writeMain` rewrites the entire file on every compile. For
  100s-of-ms latency at large project sizes we'll want to skip the
  write when the source is unchanged (cheap hash compare against
  the last write). Defer until we have a benchmark.
- The bounded-poll helper in `server.test.mjs` (25ms tick, 5s
  deadline) deliberately replaces the `setInterval` deadline check
  in the previous version. Same shape, no flake risk.
