sign-in page at `/`, three-panel editor at `/editor` (file

## Approach

- SvelteKit 2 + Svelte 5 (runes), `adapter-static`,
  prerendered, `ssr=false` everywhere. No backend yet.
- `apps/web/svelte.config.js` uses `kit.typescript.config` to
  inject `extends: ../../../tsconfig.base.json` into the
  generated `.svelte-kit/tsconfig.json`, so the workspace's
  strict TS settings still apply transitively.
- `apps/web/tsconfig.json` extends `./.svelte-kit/tsconfig.json`
  (per SvelteKit convention so `$app/*`, `$env/*`, `$lib`
  alias type definitions resolve).
- `structural_checks.py` extended to accept this transitive
  base-extension, but only when `svelte.config.js` exists
  *and* mentions `tsconfig.base.json`. Any future SvelteKit
  package without that wiring still fails the check.
- `apps/web/typecheck` is `svelte-kit sync && svelte-check
  --threshold error`, replacing `tsc --noEmit` (which can't
  see `.svelte` files).
- Hand-rolled hello-world PDF at `apps/web/static/fixture.pdf`
  (599 bytes, valid xref table, viewable). `.gitignore`
  whitelists `apps/web/static/*.pdf` since `*.pdf` is globally
  ignored for LaTeX build artefacts.

## Problems encountered

DrvFs (/mnt/c) blocks pnpm install. Multiple `pnpm install`
runs failed mid-extract with `EACCES: permission denied,
rename '..._tmp_NNN' -> '...'` because Windows file watchers
(VSCode, presumably others) hold transient handles on freshly
extracted package dirs. Half-finished `_tmp_*` dirs accumulate
and can't even be `rm -rf`'d on subsequent runs.

Workaround chosen: install onto ext4 via a symlink. `setup_node.sh`
detects a `/mnt/*` checkout, hashes the checkout path, and
points `node_modules/` at `~/.cache/tex-center-nm/<hash>/node_modules`.
Combined with `node-linker=hoisted`, the layout is flat enough
that Node's ESM resolution walks correctly through the realpath
(packages live inside a real `node_modules/` dir under ext4, so
the walk-up algorithm finds transitive deps).

Initial naive symlink target had no `node_modules` directory in
its parent — Node's walk-up skipped past the cache dir entirely
(failed importing `kleur` from `@sveltejs/kit`). Fix: target
must end in `.../node_modules`, not be a renamed directory.

  EACCES wall first. Documented in PLAN's "Local toolchain"
  section.
