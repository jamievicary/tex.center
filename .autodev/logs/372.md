## Goal for this iteration

PLAN priority #1 iter B ‚ÄĒ multi-page demand-fetch. Single coherent
slice per `369b_answer.md`: swap sidecar `runCompile`'s
`targetPage: 0` ‚Üí `maxViewingPage(p)` **early**, then mount an FE
placeholder `.pdf-page` slot when the latest segment's
`lastPage===false`. The placeholder is observed by the existing
IntersectionObserver path so a scroll bumps `maxViewingPage` ‚Üí
sidecar `recompile,N+1` ‚Üí fresh segment with the new page +
possibly another `lastPage===false` ‚Üí cascade.

This closes the long-standing wedge surfaced empirically in
`369b_question.md`: every edit was running `recompile,end` and
re-shipping the whole document (20√ó/100√ó cost on long docs).

## Approach

Five touch points, four are FE-side because the sidecar swap is a
one-line change once `maxViewingPage(p)` (already extant ‚ÄĒ see
`server.ts:453` and the iter-309/M21.3 widening of
`coalescer.kickForView`) is reused for the compile call:

1. **`apps/sidecar/src/server.ts:601`** ‚ÄĒ `targetPage: 0` ‚Üí
   `targetPage: maxViewingPage(p)`. Comment rewritten to name the
   M21 / iter-372 lineage and the FE placeholder coupling that
   resolves the M15 chicken-and-egg (no page-2 canvas without
   page-2 ship; no page-2 ship without page-2 in viewport).

2. **`apps/web/src/lib/wsClient.ts`** ‚ÄĒ `WsClientSnapshot` gains
   `pdfLastPage: boolean | undefined`. `_pdfLastPage` is stamped
   from `decoded.segment.lastPage` on every `pdf-segment` frame
   and surfaced via `snapshot()`. Tri-state (`true` ‚áí cascade
   end, `false` ‚áí mount placeholder, `undefined` ‚áí legacy mode,
   placeholder stays closed).

3. **`apps/web/src/routes/editor/[projectId]/+page.svelte`** ‚ÄĒ
   initial snapshot gains `pdfLastPage: undefined`; `<PdfViewer
   src lastPage onPageChange />`.

4. **`apps/web/src/lib/PdfViewer.svelte`** ‚ÄĒ accepts `lastPage`
   prop. New `lastPageGeometry` state remembers the most recent
   real page's CSS-px width/height (A4 fallback at bootstrap).
   `syncPlaceholder()` mounts a sibling `.pdf-page
   .pdf-page-placeholder` element after the controller commit
   when `lastPage === false` and `controller.length > 0`; the
   placeholder's `data-page` is `controller.length + 1` (1-
   indexed, matching the controller's wrappers), is observed by
   the existing IntersectionObserver, and inherits the
   `lastPageGeometry` width / aspect-ratio so its viewport ratio
   is comparable to a real page.

   Critically, `removePlaceholder()` runs **before** every render
   cycle (in the `src` $effect) so the controller's
   trailing-addition path (`createWrapper(i) ‚Üí target.appendChild
   (w)`) keeps page wrappers in DOM order when a new segment
   carries one more page than the previous round. The placeholder
   is re-mounted post-commit by `syncPlaceholder()` if still
   wanted (i.e. the new segment also carries `lastPage===false`).
   This is the DOM-stability invariant 369b_answer flagged as
   load-bearing.

   A second `$effect(lastPage)` covers the `lastPage` flip
   without an accompanying src change (e.g. an `undefined ‚Üí true`
   transition that should evict any latent placeholder).

5. **`apps/web/src/lib/PdfViewer.svelte` CSS** ‚ÄĒ placeholder gets
   a subtle paper-tone background + dashed outline (`#FAF7F0
   55%` + `#1F1B16 18%` ‚ÄĒ colour-palette echoes M16.aesthetic
   draft), no box-shadow, so the user immediately sees the slot
   as "page will fill on scroll" rather than as a missing render.

## Notes for future iterations

- **Iter B‚Ä≤ ‚ÄĒ toast-format slice (still queued).** Independent
  from iter B; small enough to land in a maintenance window.
  PLAN priority #1 iter B‚Ä≤ entry retains the format text.
- **Bootstrap-cascade gold case.** Worth a dedicated iteration
  once an iter is open. Test shape (per 369b_answer):
  fresh project, type a 2-page doc, wait for page 1 render +
  placeholder for page 2, scroll the placeholder into viewport,
  await `shipoutPage=2 lastPage=true` segment, assert placeholder
  replaced. The scroll ‚Üí IO-fires ‚Üí maxViewingPage bump ‚Üí sidecar
  compile chain is multi-hop and needs careful polling timeouts.
- **DOM stability risk (flagged in 369b_answer) is mitigated by
  pre-render `removePlaceholder()`** ‚ÄĒ the placeholder is gone
  before the controller's `createWrapper(N)` appends a real
  wrapper for the same page index. PageTracker's
  `currentMax=N` carries across the transition (the ratios map
  retains the entry until a fresh IO callback overwrites it
  with the real wrapper's ratio), so no spurious bump to N+1
  occurs while the real wrapper is mounting. The next placeholder
  (for N+1) only mounts post-commit in `syncPlaceholder()`,
  after the controller has settled.
- **If supertex daemon stops emitting `[pdf-end]`**, segments
  carry `lastPage===undefined`. The FE treats that as legacy
  mode (no placeholder) ‚ÄĒ but the sidecar's `targetPage` swap
  still clamps every compile to the highest viewer page. On a
  multi-page doc the user would see only page 1 with no demand-
  fetch slot. The submodule pointer at `aaa625a` (iter 370)
  carries the chunk_writer.c emission, so this is a wire-drift
  risk worth a quick `flyctl logs | grep pdf-end` sanity check
  during deploy verification.
- **`lastSegments` replay path.** A late-joining viewer replays
  the last shipped segment(s); the `lastPage` tri-state is
  faithfully carried on the wire (the iter-370 wire-shape
  encodes it on every `PdfSegment`), so a replayed segment
  drives the placeholder identically to a fresh one.
