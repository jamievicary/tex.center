## Goal for this iteration

Implement real "viewing page N" tracking from the browser to the
sidecar. M2 left this hardcoded to page 1 and called out the
`IntersectionObserver` approach in iter 6 notes; M3's
`target_page=N` work needs the signal flowing first. Coherent slice:
a pure `PageTracker` plus `pickMostVisible` helper (DOM-free,
unit-tested), wired into `PdfViewer.svelte` via an
`IntersectionObserver`, fed back to `WsClient.setViewingPage`
through an `onPageChange` prop on the editor page.

## Approach

- New `apps/web/src/lib/pageTracker.ts` — pure logic. Two
  exports: `pickMostVisible(items)` (highest ratio wins;
  tie-break to lower page number; ratio<=0 ignored) and a
  stateful `PageTracker` that owns a page→ratio map and emits a
  page number only when the most-visible page actually changes.
- `PdfViewer.svelte` gains an `onPageChange?` prop, a single
  long-lived `IntersectionObserver` per viewer instance (reused
  across renders), and tags each rendered canvas with
  `data-page=<n>`. On every render the observer is reset and
  re-observes the freshly created canvases. Threshold is 0..1
  in 0.1 steps — gives the tracker enough granularity to break
  ties without firing on every scroll pixel.
- `+page.svelte` passes `onPageChange={(p) => client?.setViewingPage(p)}`.
  The hardcoded `setViewingPage(1)` stays as the default until
  the IO produces its first entry.
- Unit test `apps/web/test/pageTracker.test.mjs`. No DOM needed:
  the IO-driven path is covered indirectly by feeding the same
  `(page, ratio)` shape to `PageTracker.update`. Browser-side
  IO behaviour is gold-suite (Playwright) work.

## Notes for future iterations

- `PdfViewer` still tears down all canvases on every render.
  Once supertex emits per-page byte-range deltas (M3 subgoal 5),
  re-rendering only the affected pages becomes a real win — the
  `data-page` attribute now in place is the natural key for
  that rebuild.
- The `IntersectionObserver` uses the default `root: null`
  (viewport), which works because the preview `<section>` is
  the height of the viewport. If the layout changes (e.g. a
  header is added), pass `root` = the scrolling element.
- `+page.svelte` calls `setViewingPage(1)` once on mount as a
  default. Strictly redundant once a page is rendered and the
  IO fires, but harmless and avoids a race where the first
  edit lands before the IO has reported.
- Real page-tracking under a browser is gold-suite Playwright
  work (M8); today only the pure logic has automated coverage.
