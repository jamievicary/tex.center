## Goal for this iteration

Discussion mode ‚ÄĒ `287_question.md` instructs: replace M15's live
spec with the simplest possible static failing case (no editing
at all), to disambiguate whether the bug has anything to do with
editing. The human's `284_answer.md` addendum (preview never
shows >1 page even on manually-written multi-page docs) is the
load-bearing signal.

Practical impasse identified in `287_answer.md`: no seed-override
mechanism exists; `MAIN_DOC_HELLO_WORLD` is hard-coded in
`packages/protocol/src/index.ts`. The closest faithful execution
of the user's intent without implementing a seed-override
(forbidden this iteration per "No implementation beyond the
static spec") is `Ctrl+A` ‚Üí `keyboard.type(STATIC_TWO_PAGE)`,
which atomically replaces the entire document in a single
transaction ‚ÄĒ no cursor placement, no virtual-line trap, no
coalescer round shape that depends on per-keystroke timing.
From supertex / sidecar / viewer perspectives, the resulting
input is a static known-shape 2-page document.
