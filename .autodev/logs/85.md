   `playwright.config.ts`.
2. Pure helper `resolveLocalDbEnv(env)` in
   `tests_gold/lib/src/authedCookie.ts` mirrors
   `resolveLiveDbConfig`'s missing-keys shape so the fixture

   `workerInfo.project.name === "live"`. Local path: read env,
   open `createDb(url, { onnotice: () => {} })` (suppress PGlite
   NOTICE noise), no flyctl proxy. Live path unchanged.

## Problems encountered

1. **`SyntaxError: Cannot use 'import.meta' outside a module`
   on first run.** Playwright transpiles globalSetup as CJS;
   that file imports `localDb.ts`, which used
   `import.meta.url` to default `migrationsDir`. The error
   surfaces at parse time so it fires even though
   `migrationsDir` was always going to be passed. Fix:
   `migrationsDir` is now a required input on `startLocalDb`.
   The `.mjs` gold test derives it from `import.meta.url`
   (works in true ESM); globalSetup derives it from
   `__dirname` (works in Playwright CJS).
