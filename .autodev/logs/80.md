## Goal for this iteration

Refactor iteration (`N % 10 == 0`). Survey turned up an unambiguous
duplication: `apps/sidecar/test/server{Blobs,CreateFile,DeleteFile,
RenameFile,UploadFile}.test.mjs` each inline two identical helpers —
`waitFor(check, label, frames)` and `bootClient(app)` — totalling
~25 lines × 5 files of pure copy-paste. Each new sidecar WS test
case has been growing the duplication. Extract to a shared
`apps/sidecar/test/lib.mjs` and import.

## Actions

- New `apps/sidecar/test/lib.mjs`: exports `waitFor` (verbatim) and
  `bootClient(app, projectId)` (takes `projectId` as an argument
  instead of capturing from caller closure; returns
  `{ ws, frames, clientDoc, text }` — the `text` field was only
  set by `serverBlobs.test.mjs`'s copy, but exposing it
  unconditionally costs nothing and makes the helper uniform).
- Each of the 5 test files now imports `bootClient, waitFor` from
  `./lib.mjs` and passes the local `projectId` const at the call
  site. Local helper definitions removed. Imports of
  `WebSocket`/`ws`, `Y` from `yjs`, and `decodeFrame` from the
  protocol package are dropped from the four files where the
  helper was their only consumer; kept in `serverBlobs.test.mjs`
  which uses them outside the bootClient too (lines 204/207/211
  there assemble a second WS by hand for the `beta` project's
  no-hydration path).

No production code touched. `server.test.mjs` was a near miss —
it uses an interval-based `waitForCondition` rather than the
poll-loop `waitFor` and only one call site, so consolidating it
would require unifying two slightly different shapes for a single
caller. Skipped.
