## Goal for this iteration

Fix the gold-runner's live spec which iter-165 just wired in:
every `live` Playwright case failed at the `authedPage` worker
fixture's `startFlyProxy` call with `flyctl: no access token
available. Please login with 'flyctl auth login'`. Iter-165's
loader read DB password / signing key / user id from `creds/`
but never threaded a Fly auth token, so the very first spec to
exercise the live path couldn't even open the Postgres tunnel.

## Diagnosis

`tests_gold/cases/test_playwright.py::_load_live_creds` produced
`TEXCENTER_LIVE_DB_PASSWORD`, `SESSION_SIGNING_KEY`,
`TEXCENTER_LIVE_USER_ID`. The fixture's `startFlyProxy` invokes
`flyctl proxy 5435:5432 -a tex-center-db`, which needs auth.
`creds/fly.token` (single-line `FlyV1 fm2_…` macaroon) was the
obvious supplier; flyctl honours `FLY_API_TOKEN` in the
environment, so no extra wrapper is needed.

## Actions

- `tests_gold/cases/test_playwright.py` — `_load_live_creds` now
  reads `creds/fly.token`, strips it, fails with a named-file
  message if empty, and exports it as `FLY_API_TOKEN` in the
  Playwright subprocess env. Matches the iter-165 contract of
  failing loudly on missing creds rather than skipping.
- `tests_normal/cases/test_gold_runs_live_specs.py` — added
  `fly.token` and `FLY_API_TOKEN` to the needle list, locking
  the new contract so future drift surfaces in normal tests.
