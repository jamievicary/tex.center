## Goal for this iteration

Close M7.1.3.2.c: the auth-aware redirect from `/` (the white
sign-in page) to `/projects` for signed-in visitors is bypassed in
production because `+layout.ts` defaults to `prerender = true`,
which serves a static HTML file from the edge without invoking the
SvelteKit server hook. Local Vite dev SSRs unconditionally and
masks the bug.

## Approach

Drop the layout-level `prerender = true` default and have every
concrete page opt out explicitly. `/projects` and
`/editor/[projectId]` already do this; the only missing piece was
`+page.ts` for `/`. New file `apps/web/src/routes/+page.ts` sets
`prerender = false; ssr = false`, matching the pattern of the other
authenticated pages. Layout retains `ssr = false` (SPA shell) so the
client-side runtime still drives navigation.

Added `test_landing_sign_in.test_prerender_disabled` to guard the
invariant: if `+page.ts` is ever deleted or flipped back to
prerender, the unit test screams. Pattern lifted from
`test_healthz_route.test_prerender_disabled`.

No application code change beyond the two route-config files;
behaviour change is purely "the hook now runs for GET `/` in prod".
The redirect logic in `routeRedirect` itself is unchanged and
already unit-tested.
