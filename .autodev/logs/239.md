## Goal for this iteration

Add a local Playwright regression lock for the iter 238 M13.2(a)
SSR-seed placeholder. Iter 238 landed `<pre class="editor-seed">`
inside `.editor` when `+page.server.ts` returns
`seed = { name, text }` (fresh projects with no
`machine_assignments` row). GT-6 (live) failed during iter 238's
gold run, but per the harness order (agent → tests → commit →
push) iter 238's gold ran against the *previously deployed* iter
237 web image — the SSR-seed code was not yet on live during that
test. Live `PUBLIC_TEXCENTER_ITER` is now `238`, so iter 239's
gold run will be the first to exercise the new code on live.

Independently, the local suite has no coverage for the seed
placeholder: a future iteration could regress the `data.seed`
branch in `+page.svelte` or the no-assignment gate in
`+page.server.ts` and the local suite would stay green. A small
local spec — fresh project → `goto /editor/<id>` → assert
`.editor-seed` is in DOM and carries the canonical
`documentclass` sentinel — closes that gap and makes the
mechanism testable without a live deploy round-trip.

## Approach

- New local Playwright spec `editorSeedPlaceholder.spec.ts`.
  Worker-scoped `db` fixture mints a fresh project (no machine
  assignment exists locally — the local webServer has no Fly
  Machines API target, so no row is ever written). Cleanup deletes
  the project row in `afterEach`.
- Assert `.editor-seed` is `toBeVisible()` and contains
  `documentclass` — proves the `data.seed` branch rendered.
- Assert `.cm-content` is *not* present yet — proves the
  placeholder is structurally distinct from the real CodeMirror
  mount (the live-spec invariant iter 238 leaned on).
- Local-only: `test.skip(testInfo.project.name !== "local", …)`
  in `beforeEach`. The live `cleanupLiveProjectMachine` teardown
  is not needed here because no Machine is ever spawned by the
  local webServer.

## Actions

- Added `tests_gold/playwright/editorSeedPlaceholder.spec.ts`
  (local-only). Worker `db` fixture mints a fresh project, the
  spec navigates `/editor/<id>`, asserts `.editor pre.editor-seed`
  is visible and contains both `\documentclass{article}` and
  `Hello, world!`, and asserts `.editor .cm-content` has count 0.
  `afterEach` deletes the project row.
