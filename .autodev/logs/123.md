## Approach

- `apps/sidecar/Dockerfile`: change the runtime `SIDECAR_COMPILER`
  default from `supertex` to `supertex-daemon`. `SUPERTEX_BIN`
  already points at `/opt/supertex/build/supertex`, and that
  binary supports both `--once` and `--daemon DIR` (gold test
  `supertexDaemonReal.test.mjs` proves the latter). The Dockerfile
  already runs `make -C vendor/supertex -j all`, which builds
  `supertex` plus `supertex_daemon`; the daemon compiler invokes
  the same `supertex` ELF with `--daemon`, so no new artefacts
  are needed in the runtime layer.
- `tests_normal/cases/test_sidecar_dockerfile.py`: tighten the
  regex from the word-boundary form (which matched both
  `supertex` and `supertex-daemon`) to the literal
  `SIDECAR_COMPILER=supertex-daemon`, so a future accidental
  revert fails this test loudly. Update the docstring.
- PLAN: update Current focus and Live caveats to reflect the new
  default and note the manual sidecar redeploy required for the
  live cutover (control-plane CI doesn't deploy the sidecar).
