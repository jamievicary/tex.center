## Goal for this iteration

M7.0.3.1 â€” custom Node entry that boots adapter-node's
`build/handler.js` alongside the iter-94 `attachWsProxy`,
replacing the default `build/index.js` CMD in
`apps/web/Dockerfile`. Without this entry, adapter-node's default
listener ignores HTTP Upgrade requests, so `wss://tex.center/
ws/project/<id>` would never reach the WS proxy.

## Approach

Author the entry in TypeScript so the `attachWsProxy` import path
matches the rest of the workspace, then bundle to plain JS for the
runtime image. `./handler.js` is marked external so the bundle
sits at `build/server.js` and resolves the adapter-node sibling at
runtime â€” no source duplication and no extra COPY in the
Dockerfile.

Bootstrapping logic split into a pure `boot()` function so the
WS-proxy + handler wiring can be tested without running the
file-level SIGTERM handler / `process.exit`.
