## Goal for this iteration

Add `/healthz` to `apps/web` and wire `[[http_service.checks]]` in
`fly.toml`. Iter 44 flagged this: with M6.2 deploy-on-push live but
no health route, Fly has no readiness signal — a deploy that boots
into a broken state (bad env, port mismatch, crash-on-startup) only
surfaces when a real user request 502s. M6.3 (custom domain) is the
nominal next slice but is mostly out-of-tree (Cloudflare API + DNS),
so a small in-tree win first is the right ordering. Keep the route
to liveness-only — no DB check, because Fly's readiness probe failing
on a transient DB outage would scale every machine out and serve all
traffic from nowhere; DB health belongs on a separate `/readyz`-style
endpoint if/when we add one.

## Approach

- New `apps/web/src/routes/healthz/+server.ts` exporting `GET` that
  returns `{ok: true, protocol: "tex-center-web-v1"}` with
  `Cache-Control: no-store`. No DB touch. Liveness-only — Fly only
  needs "process is answering HTTP" to mark a Machine ready.
- `fly.toml` gains a single `[[http_service.checks]]` block at
  `/healthz`. Modest interval / grace_period so a wedged deploy is
  caught within a minute or two but we don't hammer the route.
- Tests: new `tests_normal/cases/test_healthz_route.py` checks the
  route file exists and exports a `GET` handler returning 200 with
  the expected payload (parsing the .ts source statically — no node
  needed). Extend `tests_normal/cases/test_fly_toml.py` to assert
  the check block matches the route path.

## Notes for future iterations

- **`/readyz` with deps.** If we later want a *readiness* signal
  that does include Postgres + Tigris reachability, add a separate
  route — never fold deps into `/healthz`. Reason: Fly's HTTP
  service check is what gates traffic; if it 503s on a DB blip,
  every Machine drops out and the site is down even though every
  process is healthy.
- **Sidecar parity.** `apps/sidecar` already has a richer
  `/healthz` reporting `db` + `blobs` state — that's appropriate
  there because it's a per-project process, not the edge entry.
  Worth a one-line PLAN note if we ever wire Fly checks for
  sidecar Machines (M7) to use only a liveness subset.
- **M6.3 still next.** Cloudflare DNS work for the apex; mostly
  out-of-tree (API + DNS records), see iter 44 notes.
