## Notes for future iterations

- The Fly Machines API base URL `https://api.machines.dev/v1`
  works from inside Fly VMs too (no separate internal host
  needed), so the control plane talks to it the same way
  `flyctl` does from a dev box.
- `MachinesClient.createMachine` doesn't currently wait for the
  Machine to reach `started`. M7.1.2 will pair it with
  `waitForState(id, "started")` before returning the upstream
  to the WS proxy, otherwise the proxy will dial a Machine that
  isn't yet accepting connections on its 6PN address.
- The `internalAddress` helper validates with `^[A-Za-z0-9_-]+$`.
  Real Fly Machine IDs are 14-char lowercase hex but the regex
  is intentionally loose so we don't break on a future format
  change.
