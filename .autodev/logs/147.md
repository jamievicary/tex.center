## Findings (one-line summary)

- Authed WS upgrade returns **502 from Fly edge in ~243 ms**
  (one `via` hop, content-length 0). Control-plane proxy log:
  `hyper error: connection closed before message completed` for
  `/ws/project/<owned-id>`.
- Per-project sidecar Machines bind only to `127.0.0.1`
  (`apps/sidecar/src/index.ts:19` defaults `HOST` to
  `"127.0.0.1"`). 6PN IPv6 dial is refused immediately. That's
  the root cause.
- Secondary bug: `wsProxy.ts` `upstream.on("error")` path
  destroys the client socket without writing a 502, which is
  why ops sees a Fly-edge 502 instead of a tex-center 502 (and
  the only signal of the dial error is in Fly's proxy log, not
  ours).
- Anon `/ws/project/smoke` still returns 401; authed-but-not-
  owned `/ws/project/<bogus-uuid>` returns 403. The auth gate
  is working correctly — the breakage is strictly downstream of
  it.
