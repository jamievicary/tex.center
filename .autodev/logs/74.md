## Notes for future iterations

- **M7.0.1 engine binary.** Look at supertex's `discussion/` for
  the exact luatex fork in use. Two options outlined in the
  PLAN entry: build from patched source (cleanest, slow), or
  vendor a prebuilt ELF (quick, less reproducible). Prebuilt ELF
  is the right answer if the patched source isn't easily
  reproducible from the current upstream tree.
- **Image size.** texlive-full is ~5GB. Acceptable per GOAL.md.
  Once M7.0.3 is green we can profile a slim subset for a future
  iteration (`texlive` + `texlive-latex-extra` covers most users).
- **Build verification.** The Dockerfile is dark code today —
  it's authored but not built. M7.0.2's first `flyctl deploy
  --remote-only` is when real builder errors surface; the
  structural test only pins shape, not buildability. Expect to
  patch missing apt packages, COPY orders, or lockfile drift on
  that first remote build.
- **Sidecar `package.json` `start` script.** Currently
  `tsx src/index.ts`. Fine for prod-first; revisit only if cold
  start latency becomes a problem.
