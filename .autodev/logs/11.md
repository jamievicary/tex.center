## Goal for this iteration

`N=11` — plan-review iteration per `engineer.md` §3 maintenance
rule. Read `.autodev/PLAN.md` end-to-end and prune the verbose
historical narratives that have accumulated in closed milestones,
without losing live milestones, the supertex survey reference
data that M3.2–M3.5 will lean on, or the open-questions /
upstream-PR lists.

## Approach

Audited the plan section by section. What I pruned:

- **Closed milestone bullets M0/M1/M2.** Each had 8–14 lines of
  detail-of-implementation prose (file paths, framework choices,
  per-iter sub-bullets). That's now in iter logs 2–6 where it
  belongs; the plan only needs a one-paragraph summary so
  future-me can see at a glance what each closed milestone
  shipped.
- **Closed sub-milestones M3.0/M3.1.** Same compression.
- **"Current focus" section.** It opened with "M0–M2 closed; M3.0
  and M3.1 closed" — a verbatim duplicate of what the milestone
  list above already shows via [x]. Trimmed to just the live
  focus (M3.2 description + the today-state sentence).
- **Trailing recap paragraphs.** "Multi-iteration milestone with
  scaffolding: each M3.x leaves the default compile path
  runnable…" was restating what the [x]/[ ] structure plus the
  per-bullet prose already make obvious. Compressed to one
  sentence about the cutover plan + one sentence on browser test
  scope.

What I deliberately kept:

- The full **"Survey of `vendor/supertex` (iter 8)"** block. The
  flag list is reference data the next four iterations will read
  every time; the four "Implications for M3" points encode the
  decision to mirror through inotify rather than build a custom
  IPC channel. That reasoning needs to stay visible until M3 is
  done.
- The full M3.x sub-milestone breakdown including closed entries
  (in compressed form) — leaving M3.0/M3.1 as `[x]` rather than
  deleting them keeps the milestone numbering stable across the
  log record.
- "Local toolchain" (concrete, current), "Open questions / risks"
  (still open), "Candidate supertex (upstream) work" (live PR
  agenda).

Nothing was promoted to `.autodev/FUTURE_IDEAS.md` — no
nice-to-haves were hiding in the verbose prose, just history.
