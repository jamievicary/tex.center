## Goal for this iteration

Refactor iteration (N%10==0) that also closes the commitment in
`.autodev/discussion/207_answer.md`: move live-project bootstrap +
warm-up from the worker-scoped `sharedLiveProject` fixture into
`globalSetup.ts`, so Playwright's per-test `timeout` no longer has
to absorb cold-start tail. Drop per-test timeout 240s → 45s so
per-test budgets are diagnostic again.

The motivating problem: Playwright runs worker-scoped fixture
setup lazily inside the first requesting test, and the per-test
`timeout` is the budget covering that setup. Iter 206 hit this
("Test timeout of 30000ms exceeded while setting up
'liveProject'"); iter 207 patched it by hoisting `timeout` to
240s. That removed the immediate breakage but left every GT spec
with a 4-minute ceiling — slack that destroys the budget's
diagnostic value.

## Approach

1. New `tests_gold/playwright/fixtures/liveProjectBootstrap.ts`:
   - `bootstrapLiveProject()` resolves live DB config, starts a
     `flyctl proxy` (on port 5443 to avoid colliding with the
     per-worker proxy on 5433), creates a `projects` row, mints
     a session cookie, launches a transient Chromium against
     `https://tex.center`, navigates `/editor/<id>`, and waits for
     the first `TAG_PDF_SEGMENT` frame on the project WS — same
     "warm" condition the previous fixture asserted.
   - Returns `{ project, teardown }`; teardown reaps the Machine,
     deletes the project row, closes DB + flyProxy. Returns
     `null` if `FLY_API_TOKEN` or live creds are absent.
   - `exportProjectToEnv` / `readProjectFromEnv` ferry the
     `ProjectRow` shape (id, ownerId, name, createdAt, updatedAt)
     across the globalSetup → worker process boundary via env
     vars (`TEXCENTER_GT_PROJECT_*`).
2. `globalSetup.ts`: after dev server + pglite are up, invoke
   `bootstrapLiveProject`. On failure, tear down everything we've
   already started and rethrow (no leaked dev server). Stash
   `liveTeardown` into the returned closure so it runs before the
   dev server is killed.
3. `fixtures/sharedLiveProject.ts`: rewritten end-to-end. No more
   worker-scoped fixture; no more DB writes; no more Chromium
   warm-up. Test-scoped `liveProject` fixture reads env and yields
   a `ProjectRow`. Throws (not skips) if env is missing while
   `TEXCENTER_FULL_PIPELINE=1` — that combination means the gold
   runner intended to bootstrap a live project and globalSetup
   silently no-op'd, which is configuration breakage worth
   surfacing.
4. `playwright.config.ts`: `TEST_TIMEOUT_MS` 240_000 → 45_000.

## Notes for future iterations

- The first GT to land green on the new regime closes M7.4.x.
- GT-D's natural duration is ~35s; 45s is tight headroom. If a
  future deploy makes GT-D consistently exceed 40s, prefer
  raising it to 60s in `playwright.config.ts` rather than reverting
  to a 4-minute ceiling — the diagnostic budget is the point.
- The bootstrap proxy uses port 5443 to coexist with the worker
  fixture's 5433 proxy. If a CI host has 5443 in use, override
  via `TEXCENTER_GT_BOOTSTRAP_DB_LOCAL_PORT`.
- `liveProjectBootstrap.ts` deliberately hard-codes the live
  baseURL (`https://tex.center`) since globalSetup has no API to
  read per-project `use.baseURL` from the Playwright config. If
  the live target ever moves, both must move together.
