## Goal for this iteration

Discussion-mode iteration: answer `39_question.md` and execute the
re-scoping it asks for. The two-flag M3.5 plan
(`--ready-marker`, `--target-page=N`) is superseded upstream by
a future `--daemon DIR` mode, so the human asked to rip out the
sidecar code that was built against that contract — `featureDetect`,
`SupertexWatchCompiler`, `ShipoutSegmenter` — and replace it with
a sidecar-side PDF-stability debouncer that lets `SupertexOnceCompiler`
serve as the production compile path without any upstream
dependency.

Question + answer:
`.autodev/discussion/39_question.md` /
`.autodev/discussion/39_answer.md`.

## Approach

1. Delete the three obsolete modules and their three tests.
2. Simplify `SupertexOnceCompiler`: drop the `features` /
   `SupertexFeatures` plumbing, the `--target-page=N` emission, and
   the `--live-shipouts` arg (no longer consumed). Mirror in
   `supertexShared.ts` and the once-compiler test.
3. Reduce `SIDECAR_COMPILER` to `fixture` | `supertex` and drop the
   help-output capability probe. `defaultCompilerFactory` becomes
   sync.
4. Add `apps/sidecar/src/pdfStabilityWatcher.ts` — `awaitPdfStable`
   with injectable stat + clock + sleep. Default 200 ms window /
   50 ms cadence / 5 s ceiling. Standalone for now; not wired into
   `runCompile` because the once-compiler already returns after
   the engine exits, so the watcher would only add latency.
5. Tests via fake clock: settle on first stable window, ceiling
   fires on never-settling file, already-stable file, eventual
   settle, missing file, plus a real-fs smoke against a static PDF.
6. PLAN.md: collapse M3.3/M3.4/M3.5 into a "retired" note under M3
   and replace the upstream candidate list with the single
   `--daemon DIR` ask.
