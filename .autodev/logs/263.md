## Goal for this iteration

Discussion mode: answer `262_question.md`, which asks for an
audit of SVAR Svelte File Manager (`@svar-ui/svelte-filemanager`)
as an alternative to Option A (`@headless-tree`) for M11.

## Approach

1. Pull repo metadata + recent commits/releases/issues from
   GitHub for SVAR (`svar-widgets/filemanager`).
2. Cross-check the four explicit checks the question called out:
   OS drop-upload, drag-out download, theming, maintenance.
   Add: in-tree DnD (the headline M11.4 requirement) and a11y.
3. Compare against `@headless-tree` on each axis. Decide.
4. Write `262_answer.md`. No PLAN.md change since the decision
   holds.

## Findings

- Repo: 133 stars, 7 forks, 4 open issues, MIT, monthly point
  releases through v2.5 (2026-03). Single-team pace; well below
  `@headless-tree`'s ecosystem.
- **In-tree DnD: not implemented.** Open issue #5 (Aug 2025)
  is a user asking how to implement folder-into-folder dragging;
  no answer through v2.5. The headline M11.4 feature.
- **OS drop-upload / drag-out download:** neither documented.
  Readme lists "upload/download files" with no DnD claim.
- **Keyboard / a11y: partial and buggy.** Issue #4 (Aug 2025,
  open): "None of the keyboard shortcuts work in the navigation
  pane at the time of writing" — reproducible on official demo,
  still open eight months later.
- **Shape mismatch:** SVAR is a desktop-file-manager replica
  (sidebar + content pane + toolbar + breadcrumb). M11 wants a
  single source-tree pane. Opinionated styling + light/dark
  theme system would fight M16's Paper/Ink/Quill/Margin palette.
- **Bundle:** full UI component vs ~5 KB headless-tree core +
  tiny Svelte adapter.

## Decision

Stick with Option A (`@headless-tree`). SVAR is the wrong shape
of component and lacks the specific primitives M11 leans on
hardest. Recorded rationale in `262_answer.md`; no PLAN.md edit
needed since M11 already names `@headless-tree`.
