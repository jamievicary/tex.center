## Goal for this iteration

Fix the DrvFs Playwright `test-results/` EACCES failure that
surfaced in iter 141's gold suite (`rmdir
'/mnt/c/.../test-results'`). Playwright cleans its `outputDir`
between runs, and on `/mnt/c` that rmdir races with Windows file
handles and fails. The repo already redirects `node_modules` and
`.tools/playwright` to ext4 via $HOME/.cache symlinks; the same
treatment is needed for the Playwright test-results dir.

PLAN's "next ordinary iteration" pointer is M7.4.2 (upstream
supertex serialise/restore wire), but a green gold suite is a
prerequisite for the M8.acceptance work and this is a small
self-contained DrvFs hardening — landing it first costs ~one
iteration and unblocks subsequent gold-suite runs.

## Actions

- `tests_gold/playwright.config.ts`: compute `outputDir` from
  `process.cwd()` (the runner cd's to repo root). On `/mnt/*`,
  hash the path and redirect to
  `$HOME/.cache/tex-center-pw-results/<hash>/test-results`,
  `mkdirSync`-recursive. Off DrvFs, use the in-repo default.
  Playwright loads configs in CJS, so I avoided `import.meta` and
  derive the root from cwd instead — documented inline.
