## Goal for this iteration

Advance the M11.1c milestone (headless-tree adoption — highest
item in the active priority queue after M13.2(b).5 R1, which is
blocked on a shared blob store binding).

Sub-goal discovered on entry: the 260_answer.md library survey
claimed a `@headless-tree/svelte` adapter exists; it does not. A
quick `npm search` confirms only `@headless-tree/core` (framework-
agnostic) and `@headless-tree/react` are published. The recommendation
itself still holds — `core` is fine — but we have to write the
Svelte 5 binding ourselves. Splitting M11.1c into a prep slice
(adapter + tests, dark code) and a cutover slice (wire to UI)
fits cleanly into a single coherent iteration this side, leaving
the cutover for next iter once the adapter has a green pin.

## Approach

1. Verify `@headless-tree/svelte` is absent from the registry, and
   confirm `@headless-tree/core` is framework-agnostic enough to
   drive from a `pnpm exec tsx` test (no DOM).
2. Add `@headless-tree/core` to `apps/web/package.json`.
3. Write `apps/web/src/lib/fileTreeHeadless.ts`:
   - `buildFileItemMap(forest)` flattens `FileTreeNode[]` into
     `{ items, children }` maps keyed by item id (path), with a
     synthetic `__root__` entry.
   - `createFileTreeInstance(forest, opts)` constructs the tree
     via `createTree(...)`, plumbs state through a caller-supplied
     `onStateChange`, registers `syncDataLoaderFeature` +
     `selectionFeature`, and calls `setMounted(true)` +
     `rebuildTree()` so the instance is immediately usable from a
     pure-node context.
4. Unit test in `apps/web/test/fileTreeHeadless.test.mjs`:
   - `buildFileItemMap` shape (root, nested folders, leaf files).
   - Expand → child rows appear; collapse → disappear.
   - Initial state hydration (expanded + selected presets).
   - `onPrimaryAction` fires with the path on `primaryAction()`.
5. Wire the test into `tests_normal/cases/test_node_suites.py`.
6. Update `.autodev/PLAN.md`: M11.1c sub-divided into M11.1c-prep
   (landed) and M11.1c (cutover, pending).

### Notes from probing the library API

- `createTree` returns an instance whose `getItems()` is empty
  until `setMounted(true)` + `rebuildTree()` have run. The mount
  path is normally triggered by `registerElement(domNode)` on a
  Svelte/React mount handler; for pure-node tests we drive it
  manually. UI integration in M11.1c can keep doing the same — the
  flags are idempotent under a later `registerElement(...)`.
- `selectionFeature` adds `state.selectedItems` and tree-level
  `setSelectedItems(...)`. The current `FileTreeNode.svelte`
  `selected: string` (single-string) prop maps to the head of
  `selectedItems`; the cutover slice has to decide whether to
  expose multi-select.
- `hotkeysCoreFeature` attaches `document.addEventListener`
  unconditionally on mount; the cutover slice should only register
  it on the UI path, not in the pure adapter.

  current minimal pair.
