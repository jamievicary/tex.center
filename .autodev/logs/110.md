## Goal for this iteration

N=110 → refactor iteration (§3). Single highest-leverage
duplication in the tree: the four file-op sidecar server tests
(`serverCreateFile`, `serverDeleteFile`, `serverRenameFile`,
`serverUploadFile`) each open with ~15 lines of identical
scaffolding — `mkdtempSync`/`LocalFsBlobStore` setup, `blobStore
.put(...main.tex...)` seeding, `buildServer({ logger: false,
blobStore }) + app.listen({ port: 0, host: '127.0.0.1' })`, and
close-dance — plus they each repeat the `frames.find/filter((f) =>
f.kind === "control" && f.message.type === "file-list" && ...)`
pattern five to ten times.

## Approach

Extend `apps/sidecar/test/lib.mjs` with focused helpers:

- `makeBlobStore(label)` — `mkdtempSync` + `LocalFsBlobStore`.
- `seedMainTex(blobStore, projectId, content?)` /
  `seedFile(blobStore, projectId, name, content)`.
- `startServer(opts)` — `buildServer({ logger: false, ...opts })`
  + `listen({ port: 0, host: '127.0.0.1' })`.
- `closeClient(ws, app)` — `ws.close()` + once-close + `app.close()`.
- `isFileListFrame`, `fileListFrames`, `latestFileList`,
  `fileOpErrors` — frame predicates/filters.

Refactor the four file-op tests to consume them; semantics
identical (latest file-list usage is interchangeable with the
single-frame case on cold restart).
