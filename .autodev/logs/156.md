## Goal for this iteration

PLAN iter-156 stretch 1: write the payload-bearing live WS probe.
The existing `scripts/probe-live-ws.mjs` only verifies the
HTTP/1.1→WebSocket upgrade — it destroys the socket the instant
the upgrade event fires. That is enough to prove edge routing +
auth + dial reach the sidecar, but it does not exercise the
runtime: the bare upgrade barely allocates and would still pass
on the old 256MB Machines. The first probe that meaningfully
tests the iter-154 1024MB memory floor is one that holds the
socket open, decodes the leading frames the sidecar sends on
connect (`hello` control + initial Yjs state + `file-list`),
sends a control frame back, and confirms upstream doesn't
immediately close.

Stretch 2 of PLAN's iter-156 (M8.pw.4 activation) requires an
operator to create the test OAuth client in GCP and push a
secret via `flyctl secrets set`. Out of scope for this
iteration; deferred to iter 157.

  proxy 5435:5432 -a tex-center-db` (a daemon-style subprocess

  rule), plus exporting three creds. Iter 153 and 155 followed
  that operator recipe in-iteration; for this iteration the
  primary risk is wedging on the proxy lifecycle while also
  introducing new code, so I kept code-only and queued the live
