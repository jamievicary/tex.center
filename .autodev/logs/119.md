## Goal for this iteration

Open **M7.4** ‚Äî checkpoint blob protocol on the compiler
interface. Land the contract and the persistence-helper
primitives without yet wiring them into the runtime. With the
idle-stop hook landed iter 118, the next iteration can bind
these primitives in one focused slice.

## Approach

Three small additions held together by one shape:

- `Compiler.snapshot(): Promise<Uint8Array | null>` and
  `Compiler.restore(blob): Promise<void>` on the interface. Today
  no engine has a real impl, so all three concrete compilers
  return null / accept-and-discard. The contract is the
  load-bearing piece ‚Äî a future iteration that flips the daemon
  impl to real must change only the daemon class, not the
  interface or its consumers.
- `persistence.ts`: `projectCheckpointKey`, `persistCheckpoint`,
  `loadCheckpoint`. The key lives at
  `projects/<id>/checkpoint.bin`, deliberately **outside**
  `projects/<id>/files/` so source-file listing (used by the
  file tree) never surfaces it and so file-DELETE paths can
  never accidentally invalidate a stored checkpoint.
- `apps/sidecar/test/checkpointBlob.test.mjs` pins both: the
  no-op contract for every concrete compiler (constructed with a
  bogus binary path so we exercise snapshot/restore without ever
  spawning a child) and the persist/load round-trip with
  null/empty no-op semantics and per-project isolation.

Deliberately **not** in this slice: the wiring in
`buildServer`/`onIdle` to actually call snapshot+persist on
shutdown and load+restore on first compile. That's M7.4.1, a
clean follow-up once the primitives are settled. Bundling it
would conflate two reviewable pieces ‚Äî the protocol shape and
its consumers ‚Äî and would have to land tests where every
observable behaviour is "still null".

## Notes for future iterations

- **M7.4.1 shape.** `buildServer` already owns one
  `Compiler` per project (via `compilerFactory`). The hook is:
  on first `compile()` for a project, before calling the
  compiler, await `loadCheckpoint(blobStore, projectId)` and
  feed any non-null result through `compiler.restore()` ‚Äî this
  is the natural spot because it's the only place that needs to
  block until restore completes. On `onIdle` (before
  `app.close()`), walk each project, await `compiler.snapshot()`
  and `persistCheckpoint`. Because `snapshot()` returns null for
  every concrete compiler today, M7.4.1 lands as pure plumbing
  ‚Äî observable behaviour unchanged. Add a fake compiler that
  emits and consumes a deterministic blob to test the wiring.
- The `StubErrorCompiler` in `serverCompileError.test.mjs` does
  not implement `snapshot`/`restore`. That's fine today
  (`buildServer` never calls them), but M7.4.1 wiring will need
  to either extend the stub or guard the call. Prefer extending
  the stub ‚Äî duck-typed checks at the call site rot.
- The checkpoint key is `checkpoint.bin` (singular). If a future
  iteration needs versioned/historical checkpoints, prefer a
  separate subkey like `projects/<id>/checkpoints/<N>.bin` over
  shoehorning a version suffix onto the current key.
