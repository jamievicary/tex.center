- Simplified `apps/sidecar/src/compiler/supertexOnce.ts`: dropped
  the `features` / `SupertexFeatures` option, the
  `--target-page=N` emission, and the `--live-shipouts` arg
  (no in-tree consumer remained after the segmenter went).
- Trimmed `apps/sidecar/src/compiler/supertexShared.ts`:
  removed `shipoutsPath` from `SupertexPaths` and refreshed the
  module comment.
- Updated `apps/sidecar/test/supertexOnceCompiler.test.mjs`:
  removed the two `--target-page` capability cases; the fake
  supertex driver no longer accepts `--live-shipouts` /
  `--target-page=…`.
- `apps/sidecar/src/server.ts`: dropped the
  `SupertexWatchCompiler` / `detectSupertexFeatures` imports.
  `defaultCompilerFactory` is now synchronous; `SIDECAR_COMPILER`
  is `fixture | supertex` (was `fixture | supertex-once |
  supertex-watch`).
- Added `apps/sidecar/src/pdfStabilityWatcher.ts` —
  `awaitPdfStable(path, opts)` polls `(size, mtimeMs)` until two
  consecutive samples agree across the `windowMs` window
  (defaults 200 ms / 50 ms / 5 s ceiling). Injectable
  `statFn`/`nowFn`/`sleepFn` test seams; returns one of
  `stable | ceiling | missing`. **Not wired into `runCompile`** —
  the once-path's `compile()` returns after the engine exits, so
  the watcher would only add latency. Wires in when a streaming
  compiler (`--daemon DIR` consumer) starts returning before the
  PDF settles.
- Added `apps/sidecar/test/pdfStabilityWatcher.test.mjs`: six
  cases — first-window settle, ceiling on never-settling file,
  already-stable single-sample, eventual settle after churn,
  missing-file ceiling, real-fs smoke against a static PDF.
- `.autodev/PLAN.md`: M3 re-scoped — M3.3/M3.4/M3.5 retired into
  a note, M3.6 added for the new watcher, "Candidate supertex
  work" collapsed onto the single `--daemon DIR` ask. "Live
  caveats" and "Current focus" trimmed accordingly.
