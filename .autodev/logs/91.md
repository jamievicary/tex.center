## Approach

Read `vendor/supertex/tools/supertex_daemon.c` and
`tools/chunk_writer.c` to nail the exact wire format:

- `[<N>.out]\n` from `chunk_writer.c:171` (`%lld`, non-negative).
- `[rollback <K>]\n` from `supertex_daemon.c:1106` (`%lld`).
- `[error <reason>]\n` from `emit_error_line()`
  (`supertex_daemon.c:185–209`): reason is sanitised to
  printable ASCII `0x20..0x7e` minus `]`, truncated to 120
  chars. May be empty (`[error ]`) when `reason==NULL`.
- `[round-done]\n` from `supertex_daemon.c:1173, 1189`.

Per discussion 71_answer, anything else is a hard protocol
violation — the caller terminates the process and surfaces the
offending bytes. The parser models that as a fifth event kind
`{ kind: "violation", raw }` rather than throwing, so the
caller decides the kill policy.

## Actions

- Wrote `apps/sidecar/src/compiler/daemonProtocol.ts`:
  - `DaemonEvent` discriminated union: `shipout` | `rollback`
    | `error` | `round-done` | `violation`.
  - `parseDaemonLine(line: string): DaemonEvent` — strict
    regex match against the four shapes; anything else is a
    violation. Integer overflow guarded with
    `Number.isSafeInteger`.
  - `DaemonLineBuffer` — stateful chunk-to-events splitter on
    `\n`. Holds back trailing partial line; `flush()` returns
    a violation for any non-empty trailing data at EOF.
    Accepts `string` or `Uint8Array` (utf-8 decode).
- Wrote `apps/sidecar/test/daemonProtocol.test.mjs` covering:
  the four happy-path shapes, empty error reason, error
  reason with punctuation; ~13 negative inputs (missing
  brackets, stray whitespace, negative integers, trailing
  junk); buffer line splitting; chunk boundary mid-line;
  partial-at-EOF; empty line = violation; `Uint8Array` input;
  a mixed rollback-then-shipouts-then-round-done sequence;
  and an error-then-round-done sequence.
- Registered `test_sidecar_daemon_protocol` in
  `tests_normal/cases/test_node_suites.py`.

## Problems encountered

### TS `noUncheckedIndexedAccess` on regex capture

First pass typed `err[1]` as `string`; sidecar tsconfig
treats `RegExpExecArray` indices as `string | undefined`, so
typecheck failed. Fixed with `err[1] ?? ""` — empty reason
is already a valid wire value (`[error ]` when
`reason==NULL` upstream), so the fallback is semantically
correct, not a paper-over.
