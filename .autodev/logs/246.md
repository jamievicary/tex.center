## Goal for this iteration

Fix the false-RED pre-condition in
`tests_gold/playwright/verifyLiveDeleteProject.spec.ts`. Iter 245
landed the delete-project endpoint + UI, and the iter-245 live
gold run reported the spec failing at line 164
("machine_assignments row absent after editor open — pre-condition
failed"). The cause is that the spec waits for the seeded
`documentclass` text inside `.editor` and then immediately reads
`machine_assignments`; but since M13.2(a) (iter 238), that text is
rendered by the SSR seed placeholder (`<pre class=editor-seed>`)
which appears within hundreds of ms — well before the client WS
connects and `upstreamResolver` creates the per-project Machine +
upserts the `machine_assignments` row. The wait succeeds against
the wrong signal, so the row check fires too early.

Switching the pre-condition to a bounded poll on the
`machine_assignments` row itself is the right shape: it's exactly
what we need before exercising the destroy verb, and it's the
same kind of DB-side polling already used in this spec post-delete.

## Approach

In the `pre-condition` block of `verifyLiveDeleteProject.spec.ts`:

- Replace the `documentclass`-in-`.editor` poll (lines 139–155)
  with a bounded poll that calls
  `getMachineAssignmentByProjectId(db.db.db, project.id)` until it
  resolves non-null (or the EDITOR_CONTENT_TIMEOUT_MS budget is
  hit).
- Keep the subsequent direct read for the assertion-style failure
  message ("…pre-condition failed") so an unexpected null after
  the poll still flags clearly.
- Keep the WS-trigger navigation (`/projects` → click link) and
  the `waitForURL(/editor/<id>)`. The page does not need to be
  fully interactive; client-side WS bootstrap will create the
  Machine once mounted.


## Actions

  Kept the subsequent direct read + `expect(ma).not.toBeNull()` as

actually consumes, not on transient SSR text that pre-dates the
Machine. Next live run should either flip the spec green (delete
flow actually works end-to-end) or fail on a real assertion (6a,
6b, or 6c), which is the point of the pin.
