## Goal for this iteration

Extend `/healthz` to also report blob-store reachability,
mirroring the existing `db: { state }` pattern. Today the
sidecar reports DB up/down/absent but says nothing about whether
the configured blob store is reachable — relevant for
M6/M7 readiness probes and for catching mis-configured
`BLOB_STORE_LOCAL_DIR` early instead of at the next compile.

This adds a small `health()` method to the `BlobStore`
interface (throws on failure, returns void on success) and
implements it on `LocalFsBlobStore` as a `stat` of the root +
isDirectory check. The S3 adapter (M4.3.1) will implement it as
a HeadBucket call. `/healthz` calls it and reports
`blobs: { state: "absent" | "up" | "down", error? }`.

PLAN's "Current focus" steers toward M3.5 (upstream supertex
PRs, not actionable from inside this checkout) or multi-file
support / auth scaffolding (both bigger, premature without their
respective consumers). This is a small, useful step that reuses
the established health-probe shape.

## Approach

1. `packages/blobs/src/index.ts` — add `health(): Promise<void>`
   to the interface.
2. `packages/blobs/src/localFs.ts` — implement: `stat(root)`,
   throw "blob root not a directory" if not, propagate ENOENT
   etc. as a wrapped error.
3. `packages/blobs/test/localFs.test.mjs` — add health-up,
   missing-root, root-is-file cases.
4. `apps/sidecar/src/server.ts` — store the resolved
   `blobStore` on `app`, extend `/healthz` to call its
   `health()` and report the result. `ok` collapses to false on
   either probe failing.
5. `apps/sidecar/test/serverHealth.test.mjs` — add absent,
   up, down cases for `blobs`.

## Actions

- `packages/blobs/src/index.ts` — added `health(): Promise<void>`
  to the `BlobStore` interface with a doc comment naming the
  cheap-probe expectation.
- `packages/blobs/src/localFs.ts` — implemented `health()` as
  `stat(this.root)` + `isDirectory()` check; wraps ENOENT etc.
  in a "blob root … not accessible" message.
- `packages/blobs/test/localFs.test.mjs` — three new cases:
  health on a fresh tmpdir resolves; health on a non-existent
  subdir rejects with `/not accessible|ENOENT/`; health on a
  path that is a file rather than a directory rejects with
  `/not a directory/`.
- `apps/sidecar/src/server.ts` — `/healthz` now also reports
  `blobs: { state, error? }` by calling `blobStore.health()`
  when one is configured. `ok` now goes false if either probe
  is `down`.
- `apps/sidecar/test/serverHealth.test.mjs` — extended case 1
  to assert `blobs: { state: "absent" }`; added case 4 (probe
  succeeds, called once per request, called again on second
  request) and case 5 (probe throws → `down` + error).
- `.autodev/PLAN.md` — Live caveats note updated to mention the
  new blob probe and the M4.3.1 obligation to wire it for S3.
