## Actions

  PLAN, recent logs.
- Wrote `269_answer.md`. Confirmed root cause analysis
  (`target.replaceChildren()` at line 81 of PdfViewer + serial
  page render under await). Refinements:
  1. All-or-nothing swap, not per-page, to avoid the subtler
     "page 1 fades in while page 3 still rendering" flash.
  2. Don't try to abort in-flight fades; *commit instantly* on
     new render (snapshot opacity, drop old canvas, clear
     transition class) before next off-DOM render starts.
  3. Geometry change: animate wrapper width/height same duration.
  4. IntersectionObserver target = wrapper, not canvas — survives
     fades, no more `tracker.reset()` between renders.

  M11.1c slips.

  fade-controller. No pin-RED-first — the defect is visual,
