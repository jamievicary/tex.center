## Goal for this iteration

Fix the M8.pw.4 gating spec (`verifyLiveFullPipeline.spec.ts`) that
has been failing in gold for two consecutive iters (178, 179).
Root cause: iter 177's no-flash editor fix gated `.cm-content`
mount on Yjs hydration, which for a fresh-project run requires a
cold-start Fly Machine. The spec's 30s `.cm-content` wait was
sized for the pre-fix world where CodeMirror always mounted
immediately. GT-A on the same code path uses 60s; the
cold-start TCP-probe budget elsewhere is closer to 120s.
Reused-pipeline variant (warm machine) was unaffected and stayed
green throughout, which localised the diagnosis.

N=180 is normally a refactor cron iter (§3 maintenance). Deferring
that to iter 181 — fixing a two-iteration gold regression on the
canonical end-to-end spec is the higher-leverage move, and the
fix is genuinely surgical (single timeout bump + comment, no other
surface area touched).

## Approach

One-line behavioural change in `tests_gold/playwright/verifyLiveFullPipeline.spec.ts`:
bump the `.cm-content` `state: "visible"` wait from 30s to 120s,
and document why. The outer `test.setTimeout(300_000)` already
accommodates this; the inner step was the only thing too tight.

Did NOT also switch to `state: "attached"` (the way GT-A does)
because the next line is `cmContent.click()` + keyboard type, and
those want the element actually visible. The 120s budget matches
the per-project cold-start TCP-probe budget the WS-upgrade path
uses, so if we exceed it the failure is real and not a tight
timeout.

## Actions

- `tests_gold/playwright/verifyLiveFullPipeline.spec.ts`: 30s →

  iter-177 hydration gating and why the prior budget was wrong.
