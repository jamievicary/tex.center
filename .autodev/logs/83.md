## Goal for this iteration

M8.pw.1.1.b — `authedPage` Playwright fixture wiring
`startFlyProxy` (iter 82) + `mintSession` (iter 79) for the
`live` target. The fixture: starts flyctl proxy to
`tex-center-db`, opens a Drizzle handle via the proxy, mints a
session for a configured user, attaches the `tc_session` cookie
to a new browser context, yields the page, deletes the row on
teardown.

Side artefact: a small pure-helpers module
`tests_gold/lib/src/authedCookie.ts` exporting
`buildSessionCookieSpec` and `resolveLiveDbConfig`. Keeps the
fixture itself thin glue and gives us something testable
without a real flyctl or live DB.

## Approach

`resolveLiveDbConfig(env)` requires `TEXCENTER_LIVE_DB_PASSWORD`,
`SESSION_SIGNING_KEY`, and `TEXCENTER_LIVE_USER_ID`; defaults
fill in DB user/name/port/app. `buildSessionCookieSpec` returns
the Playwright cookie object matching the server's
`formatSetCookie` contract (Path=/, HttpOnly, SameSite=Lax,
Secure in prod, expires = unix seconds).

The fixture (worker-scoped `liveDb`, test-scoped `authedPage`)
calls `test.skip(true, missing-env-message)` when env is
missing, so unwired local runs and the default gold suite stay
green.

The fixture is **scaffolding**: no spec uses it yet. M8.pw.1.2
wires the first wave of authed specs onto it. Verification this
iteration is the pure-helper unit test + a smoke `import()` of
the fixture module to catch top-level load-time errors.
