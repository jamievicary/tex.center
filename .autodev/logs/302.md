## Goal for this iteration

M20.1 ‚ÄĒ two-stage idle timer in the sidecar. Active-queue #2 (after
M22 wire-message debug toasts), but the iter-301 plan-review note
flagged M20.1 as the smaller self-contained slice: replace the single
`idleTimeoutMs`/`onIdle` knob with separate `suspendTimeoutMs` (default
5 s) + `stopTimeoutMs` (default 300 s) stages, each with its own
handler. No web-tier work, no shared-blob-store work (that's M20.2).
The timers fire independently when `viewerCount` hits zero; first
viewer add clears both.

Env wiring in `apps/sidecar/src/index.ts`: `SIDECAR_SUSPEND_MS` (default
5 000) drives the suspend stage with `buildSuspendSelfFromEnv`-derived
handler (current `createIdleHandler` semantics: best-effort POST
/suspend, rearm on both success and failure); `SIDECAR_STOP_MS` (default
300 000) drives a new stop stage that closes the app and exits cleanly
‚ÄĒ the "park in `stopped`" path that the suspend-failure path used to
take eagerly (R2, iter 267) but now defers to the longer timer.

The tests assert each timer fires at its own boundary; checkpoint
persistence stays on the suspend stage (so cold-storage has the latest
snapshot before the VM freezes) and additionally runs on the stop
stage (state may have drifted post-resume).

## Actions

- `apps/sidecar/src/server.ts`: replaced single `idleTimeoutMs`/
  `onIdle` knob with two pairs ‚ÄĒ `suspendTimeoutMs`+`onSuspend` and
  `stopTimeoutMs`+`onStop`. Two timers (`suspendTimer`, `stopTimer`)
  arm together on `viewerCount‚Üí0` and on cold boot; both clear on
  any viewer connect via `clearIdleTimers()`. Each handler runs
  `persistAllCheckpoints()` before firing; each gets a `rearm`
  callback that re-arms ONLY its own timer (the other's deadline is
  fixed from the idle moment, by design ‚ÄĒ the stop stage is a
  failsafe and shouldn't reset on suspend retries).
- `apps/sidecar/src/index.ts`: replaced `createIdleHandler` with
  `createSuspendHandler` + `createStopHandler`. Suspend handler is
  the iter-267 R2 behaviour (best-effort POST `/suspend`, rearm on
  success post-resume *or* on failure, never close the app, never
  exit). Stop handler is the path to `stopped` (close the listener,
  exit 0). `main()` reads `SIDECAR_SUSPEND_MS` (default 5_000) and
  `SIDECAR_STOP_MS` (default 300_000); `SIDECAR_IDLE_TIMEOUT_MS` is
  removed (clean break ‚ÄĒ single-tenant deployment).
- Test files: `idleSuspend.test.mjs` split into suspend-handler and
  stop-handler cases (re-entrancy, no-creds path, rearm contract,
  single-shot stop). `serverIdleStop.test.mjs` adds case 6 (stop-
  alone fires), case 7 (suspend-fires-repeatedly-via-rearm while
  stop fires exactly once at its own boundary), case 8 (viewer
  re-connection cancels both timers). `serverCheckpointWiring.test.
  mjs` renamed the two `idleTimeoutMs`/`onIdle` callers to
  `suspendTimeoutMs`/`onSuspend`.
- `apps/web/src/lib/server/upstreamFromEnv.ts`: comment pointer
  updated from `createIdleHandler` to `createSuspendHandler` and a
  one-clause note about the stop stage being the cold-storage path.

## Notes for future iterations

- **M20.2 next.** Shared `BLOB_STORE` binding on the web tier and
  sidecar; the sidecar already persists supertex checkpoints to its
  blob store; M20.2 widens that to source + latex compilation
  artefacts (.aux, .log, etc ‚ÄĒ *not* supertex outputs, per user
  edit to PLAN.md) and unblocks `verifyLiveGt6LiveEditableState\
  Stopped`. The persist-before-handler hook is already in place for
  both stop and suspend stages, so M20.2 just needs to broaden what
  gets serialised, not where.
- **Suspend handler no longer exits on no-creds (local dev).** The
  pre-iter-302 behaviour was: with `suspendSelf: null`, the idle
  handler closed the app and exited 0. That's now the stop stage's
  job exclusively. Local dev sidecars without Fly creds will log a
  warning at the (default 5 s) suspend tick and exit at the
  (default 300 s) stop tick instead of the previous single 10-min
  idle exit. If this trips up dev workflow, override
  `SIDECAR_STOP_MS` upward.
- **Rearm semantics**: each rearm only re-arms its own timer.
  Considered making suspend's rearm also reset the stop timer (so
  a successful post-resume cycle yields a fresh idle window),
  decided against it ‚ÄĒ in practice the viewer that woke the VM
  immediately calls `noteViewerAdded` which clears both timers, so
  the rearm path only matters for the wake-without-viewer edge
  case. Letting stop's monotonic deadline stand is simpler and
  still correct (the failsafe fires within stopTimeoutMs of
  monotonic-time-of-idle-moment).
- **M20.3 timing override.** User-added PLAN note flags the 6 min
  end-to-end gold spec as too slow; the env vars are already the
  override mechanism ‚ÄĒ set `SIDECAR_STOP_MS=2000` or similar in
  the spec's Machine env. No code change needed.
