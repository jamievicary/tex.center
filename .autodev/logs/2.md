## Approach

Discovered while orienting that no Linux `node` is installed in
this WSL2 environment — only `/mnt/c/Program Files/nodejs/node.exe`
(Windows v24.12.0) is reachable from bash. Running TypeScript's
`tsc --noEmit` would either need Linux Node (not installed; apt
needs sudo password we don't have) or a WSL-side install via nvm
(doable, but a meaningful side-quest in its own right). Rather
than bundle a toolchain installer with the scaffolding (and risk
either breaking the iteration or growing it past the "coherent
slice" line), this iteration:

1. Lands the workspace files exactly as `tsc` would consume them
   (so the typecheck wires up in one line later).
2. Replaces `tests_normal/run_tests.sh` with structural checks
   driven by Python (JSON validity, workspace coherence, required
   files & fields, cross-package `workspace:*` references). These
   are the same kinds of breakage `tsc` would catch around the
   workspace's *shape*; what's missing is type-level checking of
   `.ts` source, which is fine because the source is currently
   one-line stubs.
3. Records the toolchain decision in PLAN under "Local toolchain"
   so the next person to touch this can pick it up cold.
